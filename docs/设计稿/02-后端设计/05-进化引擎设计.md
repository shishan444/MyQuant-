# 05 - 进化引擎设计

## 1. 概述

进化引擎是 MyQuant 策略优化的核心模块,负责通过遗传算法对交易策略 DNA 进行迭代优化。引擎协调种群管理、适应度评估、选择、交叉、变异和早停机制,以自动搜索高质量的量化交易策略。

**核心源文件**:
- `core/evolution/engine.py` - 主循环与早停检查
- `core/evolution/operators.py` - 变异算子与交叉
- `core/evolution/population.py` - 种群初始化
- `core/evolution/diversity.py` - 多样性维护

---

## 2. 整体架构

```
EvolutionEngine.evolve()
    |
    +-- 1. init_population()         初始化种群
    +-- 2. EarlyStopChecker           早停检查器
    |
    +-- [逐代循环 gen = 1 .. max_generations]
        |
        +-- 3. evaluate_fn(dna)       适应度评估 (外部注入)
        +-- 4. EarlyStopChecker.check() 早停判定
        +-- 5. Selection              精英选择
        +-- 6. Crossover              交叉产生子代
        +-- 7. Mutation               变异 (自适应强度)
        +-- 8. Fresh Blood            注入新鲜血液
        +-- 9. Diversity Check        多样性维护
        +-- 10. Trim to population_size
```

## 3. EvolutionEngine 主循环

### 3.1 构造参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `target_score` | float | 80.0 | 目标分数,达到后触发早停 |
| `template_name` | str | "profit_first" | 评分模板名称 |
| `population_size` | int | 15 | 种群大小 |
| `max_generations` | int | 200 | 最大进化代数 |
| `patience` | int | 15 | 停滞容忍代数 |
| `decline_limit` | int | 10 | 下降容忍代数 |
| `elite_ratio` | float | 0.4 | 精英保留比例 |
| `leverage` | int | 1 | 杠杆约束 |
| `direction` | str | "long" | 方向约束 (long/short/mixed) |
| `timeframe_pool` | list | [] | 可用时间框架池 (MTF) |

### 3.2 evolve() 方法签名

```python
def evolve(
    self,
    ancestor: StrategyDNA,         # 祖先策略
    evaluate_fn: Callable,         # 评估函数: DNA -> float (0-100)
    on_generation: Optional[Callable] = None,  # 回调: (gen, best, avg) -> None
) -> Dict:
    # 返回: { champion, champion_score, history, stop_reason, total_generations }
```

### 3.3 每代执行流程

**第一代之前的准备**:
1. 调用 `init_population()` 生成初始种群
2. 创建 `EarlyStopChecker` 实例
3. 初始化 `champion`、`champion_score`、`stagnation_count`

**每代的执行步骤**:

1. **约束强制执行**: 遍历种群中所有个体,强制设置 `leverage`;若 `direction != "mixed"` 则强制设置 `direction`

2. **评估**: 对每个个体调用 `evaluate_fn(dna)` 获取分数,按分数降序排列

3. **记录历史**: 记录当前代的 `best_score` 和 `avg_score`

4. **追踪冠军与停滞**: 若 `best_score > champion_score` 则更新冠军并重置 `stagnation_count`;否则 `stagnation_count += 1`

5. **回调通知**: 若提供 `on_generation`,调用 `on_generation(gen, best_score, avg_score)`

6. **早停检查**: 调用 `EarlyStopChecker.check()`,若返回 `"stop"` 则立即返回结果

7. **选择**: 保留精英, `elite_count = max(2, int(population_size * elite_ratio))`

8. **交叉与变异**: 从精英中随机选取双亲进行交叉,然后对子代施加变异

9. **新鲜血液注入**: 调用 `inject_fresh_blood()` 添加 1-2 个随机个体

10. **多样性维护**: 调用 `check_and_maintain_diversity()` 检查并替换重复个体

11. **裁剪**: 将种群裁剪到精确的 `population_size`

## 4. EarlyStopChecker 早停机制

### 4.1 四条早停规则

| 规则 | 条件 | 说明 |
|------|------|------|
| `target_reached` | `best_score >= target_score` 且 `generation >= min_generations` | 达到目标分数 (需经过最少代数以避免过早退出) |
| `stagnation` | 连续 `patience`(默认15) 代无超过 `min_improvement`(默认0.5) 的提升 | 种群停滞 |
| `decline` | 连续 `decline_limit`(默认10) 代最佳分数下降 | 性能退化 |
| `max_generations` | `generation >= max_generations` | 达到最大代数 |

### 4.2 检查逻辑

```python
def check(self, current_best: float, generation: int) -> Tuple[str, str]:
    # 1. 目标达成 (需 >= min_generations,默认20)
    if current_best >= target_score and generation >= min_generations:
        return ("stop", "target_reached")

    # 2. 停滞检查
    improvement = current_best - self.best_score
    if improvement < min_improvement:
        no_improve_count += 1
    else:
        no_improve_count = 0
    if no_improve_count >= patience:
        return ("stop", "stagnation")

    # 3. 退化检查
    if current_best < self.best_score:
        decline_count += 1
    else:
        decline_count = 0
    if decline_count >= decline_limit:
        return ("stop", "decline")

    # 4. 最大代数
    if generation >= max_generations:
        return ("stop", "max_generations")

    # 更新最佳分数
    if current_best > self.best_score:
        self.best_score = current_best
    return ("continue", "")
```

**重要细节**: `best_score` 的更新发生在所有检查之后。这意味着退化检查的基准是历史最高分,而非上一代的分数。停滞计数和退化计数是独立跟踪的。

---

## 5. Selection 选择机制

**策略**: 精英保留法

- 按分数降序排列所有个体
- 保留前 `elite_count = max(2, int(population_size * elite_ratio))` 个个体
- 默认 `elite_ratio = 0.4`,即种群大小 15 时保留 6 个精英

被淘汰的低分个体不会进入下一代。

## 6. Crossover 交叉算子

### 6.1 交叉策略

`crossover(parent_a, parent_b) -> child` 的基因组合规则:

| 基因类型 | 来源规则 |
|----------|----------|
| 入场信号 (entry_trigger + entry_guard) | 来自父本 A |
| 出场信号 (exit_trigger + exit_guard) | 来自父本 B |
| 逻辑基因 (logic_genes) | 随机选择 A 或 B |
| 风险基因 (risk_genes) | 随机选择 A 或 B |
| 执行基因 (execution_genes) | 来自父本 A |

### 6.2 MTF 层交叉

当双亲均为多时间框架策略时:

1. 按位置对应交叉各层 (`zip(parent_a.layers, parent_b.layers)`)
2. 每层独立应用与基线相同的交叉规则: 入场信号来自 A 层,出场信号来自 B 层,逻辑基因随机选择
3. `cross_layer_logic` 从双亲中随机选择
4. 子代保持与父本相同的时间框架结构

当仅一方有层结构时,子代直接继承该方的层结构。

### 6.3 边界情况

- 若父本无入场信号,使用 `a_genes[0]` 作为兜底
- 若父本无出场信号,使用 `b_genes[-1]` 作为兜底
- 若双方信号列表均为空,直接使用父本 A 的全部信号

## 7. Mutation 变异算子池

### 7.1 七种变异算子

| 算子 | 函数 | 作用范围 |
|------|------|----------|
| 参数变异 | `mutate_params` | 随机调整指标参数 (在合法范围内) |
| 指标替换 | `mutate_indicator` | 替换为同类别的替代指标 |
| 逻辑变异 | `mutate_logic` | 翻转 AND/OR 逻辑运算符 |
| 风险变异 | `mutate_risk` | 调整止损/止盈/仓位参数 |
| 增加信号 | `mutate_add_signal` | 添加随机守卫信号 |
| 移除信号 | `mutate_remove_signal` | 移除随机守卫信号 |
| 跨层逻辑 | `mutate_cross_logic` | 翻转 MTF 跨层 AND/OR 逻辑 |

其中 `mutate_cross_logic` 仅在 `timeframe_pool` 包含多个时间框架时加入变异池。

### 7.2 _pick_signal_pool() 目标选择

所有针对信号基因的变异算子都通过 `_pick_signal_pool(data)` 选择操作目标:

```
目标池数量 = 1 (基线) + len(layers) (各MTF层)
随机选择范围 = [0, n_targets - 1]
  - 选择 0: 操作基线 signal_genes
  - 选择 i (i > 0): 操作 layers[i-1].signal_genes
```

这确保 MTF 策略的各层信号基因有均等机会被变异。

### 7.3 各算子详细规则

**mutate_params**:
- 从目标池中随机选择一个含参数的信号基因
- 从该指标的注册参数中随机选择一个参数
- 变异幅度: `delta = +/- range_size * uniform(0.05, 0.30)`
- 结果按步长取整,并钳位到合法范围

**mutate_indicator**:
- 从目标池中随机选择一个信号基因
- 从 `INDICATOR_REGISTRY` 中获取同类别的替代指标
- 若目标是 trigger 角色,排除 `guard_only` 类指标
- 替换指标,参数使用默认值,条件随机生成

**mutate_logic**:
- 随机选择操作: `flip_entry` / `flip_exit` / `flip_both`
- 随机选择目标层: 基线逻辑基因或某个 MTF 层的逻辑基因
- 翻转指定层的 AND/OR

**mutate_risk**:
- `stop_loss`: 范围 [0.005, 0.20],步长 0.005,50% 概率变异
- `position_size`: 范围 [0.10, 1.0],步长 0.05,50% 概率变异
- `take_profit`: 30% 概率变异;若当前为 None,则设为 `stop_loss * uniform(1.5, 3.0)`;否则在现有基础上浮动

**mutate_add_signal**:
- 智能选择角色: 优先添加缺失的守卫类型 (entry_guard 或 exit_guard)
- 随机选择指标,随机参数和条件
- 仅添加守卫角色,不会添加新的 trigger

**mutate_remove_signal**:
- 仅移除守卫信号 (guard),绝不移除触发信号 (trigger)
- 从目标池中随机选择一个守卫信号并删除

**mutate_cross_logic** (MTF 专用):
- 仅在 `dna.is_mtf == True` 时生效
- 翻转 `cross_layer_logic`: AND -> OR 或 OR -> AND

## 8. 自适应变异强度

引擎根据 `stagnation_count` (连续无提升代数) 动态调整每次交叉后的变异数量:

| 停滞程度 | 变异数量 | 权重分布 |
|----------|----------|----------|
| `stagnation_count > 8` (严重停滞) | 从 [2, 3, 4] 中选择 | [30, 45, 25] |
| `stagnation_count > 4` (中度停滞) | 从 [1, 2, 3] 中选择 | [25, 45, 30] |
| 正常状态 | 从 [1, 2, 3] 中选择 | [50, 35, 15] |

**设计意图**: 种群停滞时加大变异幅度以跳出局部最优;正常时偏向少量变异以保留优良特征。

### 8.1 变异常规流程

```python
child = crossover(p1, p2)
n_mutations = random.choices([1,2,3], weights=...)  # 根据停滞程度
for _ in range(n_mutations):
    mut = random.choice(mutation_pool)
    child = mut(child)
```

交叉失败时,用 `create_random_dna()` 生成随机个体作为兜底。

---

## 9. 新鲜血液注入

每代通过 `inject_fresh_blood()` 注入随机个体:

- 注入数量: `max(0, population_size - len(current_population))`
- 实际上每代最多注入 1-2 个 (因为子代数量为 `population_size - elite_count - 2`)
- 新个体通过 `create_random_dna()` 生成,遵守任务级约束 (leverage, direction, timeframe_pool)
- 时间框架和交易对继承自现有种群第一个个体

**目的**: 引入全新的基因组合,防止种群过早收敛。

## 10. 多样性维护

### 10.1 基因签名

`_gene_signature(dna)` 为每个个体生成唯一标识字符串:

```
格式: indicator_name,p=first_param_value:condition_type:role | ... | lev:N | dir:X
示例: RSI,p=14:lt:entry_trigger|EMA,p=20:cross_above:exit_trigger|lev:3|dir:long
```

签名包含: 指标名、首个参数值、条件类型、角色、杠杆和方向。

### 10.2 多样性计算

`compute_diversity(population)` 计算种群多样性比率:

```
diversity = unique_signatures / population_size
```

范围: 0.0 (全部相同) 到 1.0 (全部唯一)。

### 10.3 多样性维护机制

`check_and_maintain_diversity(population, threshold=0.30)`:

1. 计算当前多样性
2. 若多样性低于阈值 (默认 0.30):
   - 统计各签名出现次数
   - 若最常见签名占比超过 30%: 执行去重替换
   - 遍历种群,遇到已见过的签名时,用 `create_random_dna()` 替换该个体
   - 仅保留每个签名的首次出现

## 11. 种群初始化

### 11.1 init_population()

```python
def init_population(
    size=15, ancestor=None, timeframe="4h", symbol="BTCUSDT",
    leverage=1, direction="long", timeframe_pool=None,
) -> List[StrategyDNA]:
```

**流程**:
1. 若提供 `ancestor`,将其作为第一个个体;否则生成随机 DNA
2. 循环生成剩余个体:
   - 从前 5 个个体中随机选择父本
   - 随机选择一种变异算子 (`mutate_params`, `mutate_indicator`, `mutate_logic`, `mutate_risk`)
   - 验证变异结果 (`validate_dna`)
   - 有效则加入种群,无效则用随机 DNA 替代

### 11.2 create_random_dna()

生成完全随机的合法策略 DNA:

**信号基因**:
- 必选: 1 个 `entry_trigger` + 1 个 `exit_trigger`
- 50% 概率: 1 个 `entry_guard`
- 40% 概率: 1 个 `exit_guard`
- 指标从注册表中随机选取 (trigger 角色排除 `guard_only` 指标)
- 参数在合法范围内随机生成

**逻辑基因**: `entry_logic` 和 `exit_logic` 各 50% 概率选 AND 或 OR

**风险基因**:
- `stop_loss`: uniform(0.01, 0.15)
- `position_size`: uniform(0.10, 0.60)
- `take_profit`: None

**验证兜底**: 若生成的 DNA 未通过 `validate_dna()`,回退到硬编码的 RSI 策略:
- 入场: RSI(14) < 30
- 出场: RSI(14) > 70

**MTF 支持**: 若提供 `timeframe_pool` 且包含多个时间框架:
- 为每个非执行时间框架创建随机 `TimeframeLayer`
- 随机选择 `cross_layer_logic` (AND/OR)

### 11.3 create_random_mtf_layer()

为 MTF 策略生成单个随机时间框架层:
- 必选 1 个 `entry_trigger` + 1 个 `exit_trigger`
- 50% 概率添加 `entry_guard`
- 独立的 `LogicGenes`

## 12. 额外变异算子 (MTF 结构)

除标准 7 种变异外,还有 3 种 MTF 结构变异算子 (当前未加入主变异池,可按需扩展):

| 算子 | 作用 |
|------|------|
| `mutate_add_layer` | 添加新的时间框架层 |
| `mutate_remove_layer` | 移除非执行时间框架层 (至少保留1层) |
| `mutate_layer_timeframe` | 变更非执行层的时间框架 |

## 14. 设计约束与注意事项

1. **变异后不验证**: 主循环中对子代的变异操作不进行 `validate_dna()` 检查,依赖变异算子本身的合法性保证
2. **种群大小不变**: 每代结束时裁剪到精确的 `population_size`,超出的个体被丢弃
3. **评估函数外部注入**: `evaluate_fn` 由调用方提供,引擎不关心评估细节 (回测 + 评分)
4. **MTF 约束**: 当 `timeframe_pool` 只有 1 个时间框架或为空时,不启用 MTF 相关功能
5. **方向 mixed 模式**: 当 `direction="mixed"` 时,不强制覆盖个体的方向基因,允许进化探索多空双向
6. **不可变性**: 每次变异和交叉都通过 `to_dict()` / `from_dict()` 创建新的 DNA 实例,不修改原始对象
