# 07 - 评分系统设计

## 1. 概述

评分系统将回测产生的原始绩效数据转化为 0-100 的标准化综合分数。系统采用四阶段流水线架构: 指标提取、归一化、模板加权和交易次数惩罚。评分结果直接作为进化引擎的适应度值,驱动策略的优化方向。

**核心源文件**:
- `core/scoring/metrics.py` - 原始指标提取
- `core/scoring/normalizer.py` - 归一化映射
- `core/scoring/templates.py` - 评分模板定义
- `core/scoring/scorer.py` - 综合评分器

---

## 2. 四阶段评分流水线

```
原始数据 (equity_curve, trades)
    |
    +-- Stage 1: compute_metrics()      提取原始指标
    |       输出: { annual_return, sharpe_ratio, max_drawdown, win_rate, calmar_ratio, total_trades }
    |
    +-- Stage 2: normalize()            归一化到 0-100
    |       输出: { dim: score_0_100 }
    |
    +-- Stage 3: ScoringTemplate        按模板权重加权
    |       输出: weighted_sum
    |
    +-- Stage 4: trade_count_penalty    交易次数惩罚
    |
    +-- 最终输出: total_score (0-100)
```

## 3. Stage 1: 指标提取 (metrics.py)

### 3.1 compute_metrics() 函数

```python
def compute_metrics(
    equity_curve: pd.Series,
    total_trades: int = 0,
    bars_per_year: int = 365 * 6,          # 默认 4h K线
    trade_win_rate: float | None = None,    # 交易级别胜率
    trade_returns: np.ndarray | None = None, # 每笔交易收益率
) -> dict:
```

### 3.2 零交易处理

若 `len(equity_curve) < 2` 或 `total_trades == 0`,所有指标返回 0.0:

```python
return {
    "annual_return": 0.0,
    "sharpe_ratio": 0.0,
    "max_drawdown": 0.0,
    "win_rate": 0.0,
    "calmar_ratio": 0.0,
    "total_trades": 0,
}
```

### 3.3 年化收益率 (annual_return)

计算方法: 基于权益曲线的总收益率,按复利公式年化。

```python
total_return = (equity_curve[-1] / equity_curve[0]) - 1
n_bars = len(equity_curve)
years = n_bars / bars_per_year
annual_return = (1 + total_return) ** (1 / max(years, 0.01)) - 1
```

- 使用 `max(years, 0.01)` 防止除零
- 计算结果可正可负

### 3.4 最大回撤 (max_drawdown)

计算方法: 基于权益曲线的累计最大值。

```python
cummax = equity_curve.cummax()
drawdown = (equity_curve - cummax) / cummax
max_drawdown = float(drawdown.min())
```

- 返回值为负数,例如 -0.60 表示 60% 的最大回撤
- NaN 检查: 若结果为 NaN,设为 0.0

### 3.5 夏普比率 (sharpe_ratio)

夏普比率的计算有两条路径,优先使用交易级别数据:

**路径 1: 交易级别夏普 (优先)**

条件: `trade_returns` 非空且长度 > 1

```python
trade_mean = np.mean(trade_returns)
trade_std = np.std(trade_returns, ddof=1)
trades_per_year = total_trades / max(years, 0.01)
sharpe = (trade_mean / trade_std) * sqrt(trades_per_year)
```

- `trade_returns` = 每笔交易的 PnL / init_cash
- 使用 ddof=1 (样本标准差)
- 年化因子: `sqrt(trades_per_year)`

**路径 2: K线级别夏普 (回退)**

条件: 路径 1 不满足,且 `total_trades >= 5`

```python
returns = equity_curve.pct_change().dropna()
mean_ret = returns.mean()
std_ret = returns.std()
sharpe = (mean_ret / std_ret) * sqrt(bars_per_year)
```

- 仅当交易次数 >= 5 时才使用 K 线级别计算
- 使用 bars_per_year 作为年化因子

**路径 3: 零值**

条件: 交易级别不可用且交易次数 < 5

```
sharpe_ratio = 0.0
```

**设计理由**: 交易级别数据更精确地反映策略质量;K 线级别的夏普在交易稀少时不可靠,因此设下限。

### 3.6 胜率 (win_rate)

**优先使用交易级别胜率**:

```python
if trade_win_rate is not None:
    win_rate = trade_win_rate
```

**回退到 K 线级别胜率**:

```python
returns = equity_curve.pct_change().dropna()
win_rate = (returns > 0).sum() / len(returns)
```

K 线级别胜率衡量的是正收益K线占总K线的比例,精确度低于交易级别。

### 3.7 卡玛比率 (calmar_ratio)

```python
if abs(max_drawdown) > 0.001:
    calmar_ratio = abs(annual_return / max_drawdown)
else:
    calmar_ratio = 0.0
```

- 年化收益率与最大回撤的绝对值之比
- 回撤接近零时设为 0.0 以避免除零

## 4. Stage 2: 归一化 (normalizer.py)

### 4.1 normalize() 函数

```python
def normalize(metric_name: str, value: float) -> float:
    # 返回: 0.0 - 100.0
```

### 4.2 各维度归一化规则

**annual_return (年化收益率)**

```
映射: [-1.0, +1.0] -> [0, 100]
公式: score = (value + 1.0) / 2.0 * 100
```

- -100% 收益 = 0 分
- 0% 收益 = 50 分
- +100% 收益 = 100 分
- 线性映射,无特殊惩罚

**sharpe_ratio (夏普比率)**

```
正值: 映射 [0, 3.0] -> [0, 100]
      公式: score = min(value / 3.0, 1.0) * 100
      上限: >= 3.0 = 100 分

负值: score = max(0, 50 + value * 10)
      例如: -1.0 -> 40, -5.0 -> 0
```

- 夏普 >= 3.0 视为优秀,直接满分
- 负夏普不会得到 0 以下的分数

**max_drawdown (最大回撤)**

```
基础: 线性映射,0%回撤=100分,50%回撤=0分
公式: score = (1.0 + value) * 100
      其中 value 为负值 (如 -0.60 表示 60% 回撤)

惩罚: 回撤 > 20% 时施加平方惩罚
      score = score * (1.0 + value)
```

- 10% 回撤: (1.0 + (-0.10)) * 100 = 90 分
- 20% 回撤: (1.0 + (-0.20)) * 100 = 80 分,无额外惩罚
- 30% 回撤: 70 * (1.0 + (-0.30)) = 70 * 0.70 = 49 分
- 50% 回撤: 50 * (1.0 + (-0.50)) = 50 * 0.50 = 25 分

平方惩罚确保大回撤被显著扣分。

**win_rate (胜率)**

```
映射: [0.4, 0.7] -> [0, 100]
公式: score = (value - 0.4) / 0.3 * 100
```

- 40% 胜率 = 0 分
- 55% 胜率 = 50 分
- 70% 胜率 = 100 分
- 低于 40% 或高于 70% 的值会被 clamp 到 [0, 100]

**calmar_ratio (卡玛比率)**

```
映射: [0, 5.0] -> [0, 100]
公式: score = min(value / 5.0, 1.0) * 100
上限: >= 5.0 = 100 分
```

**未知维度**: 返回 50.0 (中间分数)

### 4.3 范围钳位

所有维度的归一化结果都被钳位到 [0.0, 100.0]:

```python
return max(0.0, min(100.0, score))
```

## 5. Stage 3: 评分模板 (templates.py)

### 5.1 ScoringTemplate 数据结构

```python
@dataclass
class ScoringTemplate:
    name: str                    # 模板名称
    weights: Dict[str, float]    # 维度权重
    threshold: float             # 合格阈值
```

### 5.2 四种评分模板

**profit_first (盈利优先)**

| 维度 | 权重 |
|------|------|
| annual_return | 0.35 |
| sharpe_ratio | 0.25 |
| max_drawdown | 0.25 |
| win_rate | 0.15 |
| **合格阈值** | **75.0** |

侧重绝对收益,适合追求高回报的场景。

**steady (稳健型)**

| 维度 | 权重 |
|------|------|
| annual_return | 0.20 |
| sharpe_ratio | 0.35 |
| max_drawdown | 0.35 |
| calmar_ratio | 0.10 |
| **合格阈值** | **80.0** |

侧重风险调整后的稳定表现,高权重给夏普和回撤。

**risk_first (风控优先)**

| 维度 | 权重 |
|------|------|
| annual_return | 0.10 |
| sharpe_ratio | 0.30 |
| max_drawdown | 0.40 |
| calmar_ratio | 0.20 |
| **合格阈值** | **82.0** |

最大权重给回撤 (40%),最低权重给绝对收益 (10%)。合格阈值最高。

**custom (自定义均衡)**

| 维度 | 权重 |
|------|------|
| annual_return | 0.20 |
| sharpe_ratio | 0.20 |
| max_drawdown | 0.20 |
| win_rate | 0.20 |
| calmar_ratio | 0.20 |
| **合格阈值** | **80.0** |

所有维度等权重,适用于需要全面评估的场景。

### 5.3 权重校验

四种模板的权重之和均为 1.0:

- profit_first: 0.35 + 0.25 + 0.25 + 0.15 = 1.00
- steady: 0.20 + 0.35 + 0.35 + 0.10 = 1.00
- risk_first: 0.10 + 0.30 + 0.40 + 0.20 = 1.00
- custom: 0.20 * 5 = 1.00

## 6. Stage 4: 综合评分 (scorer.py)

### 6.1 score_strategy() 函数

```python
def score_strategy(
    metrics: dict,                          # compute_metrics() 的输出
    template_name: str = "profit_first",    # 模板名称
    template: ScoringTemplate | None = None, # 直接传入模板 (优先于名称)
) -> Dict:
    # 返回: { total_score, dimension_scores, template_name, threshold, raw_metrics }
```

### 6.2 零交易规则

```python
if metrics.get("total_trades", 0) == 0:
    return {
        "total_score": 0.0,
        "dimension_scores": {},
        "template_name": template.name,
        "threshold": template.threshold,
        "raw_metrics": metrics,
    }
```

零交易策略直接得 0 分,跳过所有后续计算。

### 6.3 归一化各维度

```python
dimension_scores = {}
for dim, weight in template.weights.items():
    raw_val = metrics.get(dim, 0.0)
    dimension_scores[dim] = normalize(dim, raw_val)
```

每个维度独立归一化到 0-100。

### 6.4 加权求和

```python
total = sum(
    dimension_scores.get(dim, 0.0) * weight
    for dim, weight in template.weights.items()
)
```

### 6.5 交易次数惩罚

```python
MIN_TRADES_THRESHOLD = 10
trade_count = metrics.get("total_trades", 0)
trade_factor = min(1.0, trade_count / MIN_TRADES_THRESHOLD)
total *= trade_factor
```

**惩罚规则**:

| 交易次数 | trade_factor | 效果 |
|----------|-------------|------|
| 0 | 0.0 | 零分 (已被零交易规则拦截) |
| 1 | 0.1 | 扣减 90% |
| 5 | 0.5 | 扣减 50% |
| 8 | 0.8 | 扣减 20% |
| >= 10 | 1.0 | 无惩罚 |

**设计意图**: 交易次数过少的策略,其统计指标不可靠。线性惩罚确保少量交易无法获得高分。

### 6.6 返回结果

```python
{
    "total_score": float,               # 最终综合分数 (0-100)
    "dimension_scores": {               # 各维度归一化分数
        "annual_return": float,
        "sharpe_ratio": float,
        "max_drawdown": float,
        ...
    },
    "template_name": str,               # 使用的模板名
    "threshold": float,                 # 合格阈值
    "raw_metrics": dict,                # 原始指标 (调试用)
}
```

## 7. 完整评分示例

### 7.1 示例策略

假设一个回测结果:
- annual_return = 0.85 (85%)
- sharpe_ratio = 1.8
- max_drawdown = -0.25 (25% 回撤)
- win_rate = 0.58 (58%)
- total_trades = 8

使用 profit_first 模板:

### 7.2 归一化

```
annual_return: (0.85 + 1.0) / 2.0 * 100 = 92.5
sharpe_ratio:  min(1.8 / 3.0, 1.0) * 100 = 60.0
max_drawdown:  (1.0 + (-0.25)) * 100 = 75.0, 再加惩罚: 75.0 * 0.75 = 56.25
win_rate:      (0.58 - 0.4) / 0.3 * 100 = 60.0
```

### 7.3 加权求和

```
total = 92.5 * 0.35 + 60.0 * 0.25 + 56.25 * 0.25 + 60.0 * 0.15
      = 32.375 + 15.0 + 14.0625 + 9.0
      = 70.4375
```

### 7.4 交易次数惩罚

```
trade_factor = min(1.0, 8 / 10) = 0.8
final_score = 70.4375 * 0.8 = 56.35
```

最终分数: 56.35 (未达到 profit_first 的 75 分阈值)

## 8. 设计约束与注意事项

1. **数据源优先级**: 夏普比率和胜率都优先使用交易级别数据,仅在不可用时回退到 K 线级别
2. **夏普计算门槛**: K 线级别夏普要求至少 5 笔交易,否则返回 0.0
3. **交易惩罚不可逆**: 交易次数惩罚是最后一步,直接乘到加权总分会上,无法被其他维度的高分弥补
4. **回撤惩罚非线性**: max_drawdown 超过 20% 后额外施加乘法惩罚,使大回撤策略的分数显著下降
5. **阈值仅做参考**: `threshold` 字段不会自动过滤策略,仅用于展示或由调用方判断
6. **模板不可扩展**: 新增模板需修改 `SCORING_TEMPLATES` 字典,无动态注册机制
7. **归一化对称性**: annual_return 的归一化是对称的 ([-100%, +100%]),但 sharpe_ratio 不对称 (负值区间压缩)
