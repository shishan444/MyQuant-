# 04 - 指标计算体系

## 1. 概述

MyQuant 的指标计算体系分为两层: 指标注册表 (registry) 和指标计算引擎 (indicators)。
注册表定义每个指标的元信息（参数约束、输出字段、支持的条件类型），
计算引擎负责实际的指标数值计算。

**源码位置**:
- `core/features/registry.py` - 指标定义与注册表
- `core/features/indicators.py` - 计算引擎与预计算逻辑

**底层依赖**: pandas-ta 库提供所有标准指标的计算能力。

## 2. 注册表数据结构

### 2.1 ParamDef - 参数定义

```python
@dataclass
class ParamDef:
    type: str          # "int" | "float"
    min: float         # 最小值
    max: float         # 最大值
    default: float     # 默认值
    step: float        # 步进值 (变异粒度)
```

ParamDef 定义指标的每个参数的取值范围和约束，用于:
- 进化算法的参数变异（在 min/max 范围内按 step 步进）
- 参数校验

`clamp(value)` 方法将值约束到有效范围并四舍五入到最近的 step 边界:
```python
value = max(self.min, min(self.max, value))
return round(value / self.step) * self.step
```

### 2.2 IndicatorDef - 指标定义

```python
@dataclass
class IndicatorDef:
    category: str                        # 指标类别
    params: Dict[str, ParamDef]          # 参数定义字典
    output_fields: List[str]             # 输出字段列表
    supported_conditions: List[str]      # 支持的条件类型
    guard_only: bool = False             # 是否仅用作 Guard
```

**category**: 指标所属类别，用于同类指标间的变异替换。

**output_fields**: 指标计算后产生的列字段名（不含参数部分）。
例如 MACD 的 output_fields = ["macd", "signal", "histogram"]。

**supported_conditions**: 该指标支持的条件类型列表。
例如 RSI 仅支持阈值比较 (lt, gt, le, ge)，EMA 支持价格穿越 (price_above, price_below)。

**guard_only**: 标记为 True 的指标只能作为 ENTRY_GUARD 或 EXIT_GUARD 角色，
不能作为 Trigger。这类指标通常用于过滤而非信号触发。
guard_only 指标包括: ATR, ADX, PSAR, VolumeProfile。

## 6. 滚动成交量分布: _rolling_volume_profile()

```python
def _rolling_volume_profile(
    close: pd.Series,
    volume: pd.Series,
    bins: int = 50,
    lookback: int = 60,
    which: str = "poc",
) -> pd.Series
```

VolumeProfile 的底层实现。在每个 bar 上，对过去 lookback 根 bar 的数据构建价格-成交量直方图。

### 6.1 算法步骤

```
对每个 bar (从 lookback-1 开始):
  1. 取最近 lookback 根 bar 的 close 和 volume
  2. 计算 close 的 min/max
  3. 将价格区间分为 bins 个等宽桶
  4. 将每根 bar 的成交量归入对应价格桶
  5. POC = 成交量最大的桶的中心价格
  6. Value Area:
     a. 按成交量降序排列桶
     b. 依次累加直到达到总成交量的 70%
     c. VAH = 入选桶的上界最大值
     d. VAL = 入选桶的下界最小值
```

### 6.2 性能考量

VolumeProfile 是计算开销最大的指标。每根 bar 需要一次直方图构建和排序，
时间复杂度为 O(n * (lookback + bins*log(bins)))。

因此 VolumeProfile 标记为 guard_only 且不参与默认预计算，
仅在 DNA 中显式包含时按需计算。

## 8. 批量预计算: compute_all_indicators()

```python
def compute_all_indicators(df: pd.DataFrame) -> pd.DataFrame
```

对 OHLCV DataFrame 批量计算所有默认参数的指标。

### 8.1 处理流程

```
1. 拷贝输入 df
2. 遍历 _DEFAULT_PARAMS 的每个 (指标名, 参数集合列表)
3. 对每组参数调用 _compute_indicator()
4. 仅添加 DataFrame 中尚不存在的列（避免重复计算）
5. 计算失败的指标静默跳过
6. 返回增强后的 DataFrame
```

### 8.2 调用位置

- `core/data/mtf_loader.py` 的 `load_and_prepare_df()` 函数
- 回测路径中加载数据后的指标预计算

### 8.3 性能说明

对于 10000 行 OHLCV 数据，预计算所有 44 组参数大约需要 1-3 秒。
结果 DataFrame 约增加 55 列，内存占用增加约 4-5 MB。

## 9. 列名规范

所有指标列名遵循统一的命名规范，确保 executor.py 的 `_get_indicator_column()` 能可靠查找。

### 9.1 命名规则

| 模式 | 适用指标 | 示例 |
|------|---------|------|
| `{prefix}_{param}` | 单参数单输出 | ema_20, rsi_14, atr_14 |
| `{prefix}_{p1}_{p2}_{p3}` | 多参数 | macd_12_26_9 |
| `{prefix}_{field}_{p1}_{p2}` | 多输出 | macd_histogram_12_26_9, bb_upper_20_2 |
| `{prefix}_{p1}_{p2}_{p3}_{field}` | 通道类 | kc_20_10_2.0_upper, dc_upper_20 |
| `{field}` | 无参数 | obv, cvd, ad, psar |
| `{field}_{p1}_{p2}` | 自定义 | vp_poc_50_60 |

### 9.2 具体命名约定

- 所有列名使用小写
- 下划线分隔各部分
- float 参数中的 ".0" 在布林带和 Keltner 中保留，在其他地方去除
- VolumeProfile 字段前缀为 "vp_"
- 唐奇安通道前缀为 "dc_"
- 肯特纳通道前缀为 "kc_{params}_"

### 9.3 executor.py 与 indicators.py 的命名一致性

`_get_indicator_column()` 中的列名生成逻辑必须与 `_compute_indicator()` 中的列名
完全一致。任何不匹配都会导致 ValueError，在信号生成时该基因被跳过。

两端的关键对应点:
- BB 的 std 参数: indicators.py 使用 `str(std)` (保留 .0)，executor.py 使用 `str(std).replace(".0", "")`
  (去除 .0)。这是当前代码中存在的一个潜在不一致点，但 executor 的 fallback 查找机制可以覆盖。
