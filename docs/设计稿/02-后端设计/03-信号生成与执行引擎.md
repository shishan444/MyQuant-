# 03 - 信号生成与执行引擎

## 1. 概述

executor.py 是 DNA 基因模型到可执行交易信号的桥梁模块。它将 StrategyDNA 中的基因结构转换为
pandas 布尔序列 (entry/exit signals)，供回测引擎消费。

**源码位置**: `core/strategy/executor.py`

**核心数据流**:

```
StrategyDNA + DataFrame(s) -> dna_to_signals() -> (entry_series, exit_series)
```

## 2. 条件评估: evaluate_condition()

### 2.1 函数签名

```python
def evaluate_condition(
    indicator_series: pd.Series,   # 指标值序列
    close_series: pd.Series,       # 收盘价序列
    condition: dict,               # 条件字典
    df: pd.DataFrame | None = None # 完整增强 DataFrame (部分条件需要)
) -> pd.Series:                    # 返回布尔序列
```

### 2.2 条件评估逻辑

每种条件类型对应一个布尔 Series 的计算方式:

#### 阶段 1 - 基础比较条件

**lt (小于)**:
```python
indicator_series < condition["threshold"]
```
示例: RSI < 30

**gt (大于)**:
```python
indicator_series > condition["threshold"]
```
示例: RSI > 70

**le (小于等于)**:
```python
indicator_series <= condition["threshold"]
```

**ge (大于等于)**:
```python
indicator_series >= condition["threshold"]
```

**cross_above (向上穿越阈值)**:
```python
(indicator_series.shift(1) < threshold) & (indicator_series >= threshold)
```
当前 bar 指标值 >= 阈值，且前一个 bar 指标值 < 阈值。
若未提供 threshold，返回全 False。

**cross_below (向下穿越阈值)**:
```python
(indicator_series.shift(1) > threshold) & (indicator_series <= threshold)
```
当前 bar 指标值 <= 阈值，且前一个 bar 指标值 > 阈值。

**price_above (价格在指标上方)**:
```python
close_series > indicator_series
```
无需 threshold，直接比较收盘价与指标值。
典型用于 EMA、VWAP 等趋势指标。

**price_below (价格在指标下方)**:
```python
close_series < indicator_series
```

#### 阶段 2 - 动态上下文条件

**cross_above_series (指标 A 向上穿越指标 B)**:
```python
(indicator_series.shift(1) < target_series.shift(1)) & (indicator_series >= target_series)
```
需要 condition 包含 `target_indicator` 和 `target_params`。
通过 `_resolve_target_indicator_series()` 从 df 中解析目标指标列。

**cross_below_series (指标 A 向下穿越指标 B)**:
```python
(indicator_series.shift(1) > target_series.shift(1)) & (indicator_series <= target_series)
```

**lookback_any (回溯窗口内任一满足)**:
```python
inner_signal.rolling(window=window, min_periods=1).apply(any, raw=False)
```
- `window`: 回溯窗口大小，默认 5
- `inner`: 内嵌条件字典，通过 `_evaluate_inner_condition()` 评估
- 对内嵌条件信号做 rolling 窗口内 any 检查

**lookback_all (回溯窗口内全部满足)**:
```python
inner_signal.rolling(window=window, min_periods=1).apply(all, raw=False)
```
与 lookback_any 类似，但要求窗口内所有 bar 都满足。

#### 阶段 4 - 支撑阻力条件

**touch_bounce (触及反弹)**:

direction="support" (支撑反弹):
```python
proximity = (low - line).abs() <= line.abs() * proximity_pct
above_line = close > line
next_close_up = close.shift(-1) > close
result = proximity & above_line & next_close_up
```
三个条件: (1) 最低价接近指标线 (2) 收盘价在指标线上方 (3) 下一根 bar 收盘更高

direction="resistance" (阻力反弹):
```python
proximity = (high - line).abs() <= line.abs() * proximity_pct
below_line = close < line
next_close_down = close.shift(-1) < close
result = proximity & below_line & next_close_down
```

默认 proximity_pct = 0.01 (1%)。

**role_reversal (角色转换)**:

role="support" (从阻力转为支撑):
```python
was_below = close.shift(lookback) < line.shift(lookback)
now_above = close > line
result = was_below & now_above
```
lookback 个 bar 前价格在指标线下方 (指标是阻力)，现在价格在上方 (指标变为支撑)。

role="resistance" (从支撑转为阻力):
```python
was_above = close.shift(lookback) > line.shift(lookback)
now_below = close < line
result = was_above & now_below
```

默认 lookback = 10。

**wick_touch (影线触及)**:

direction="above" (上影线触及):
```python
wick_near = (high - line).abs() <= line.abs() * proximity_pct
close_below = close < line
result = wick_near & close_below
```
最高价接近指标线，但收盘在下方。

direction="below" (下影线触及):
```python
wick_near = (low - line).abs() <= line.abs() * proximity_pct
close_above = close > line
result = wick_near & close_above
```
最低价接近指标线，但收盘在上方。

默认 proximity_pct = 0.01。

**未知条件类型**: 返回全 False 的布尔 Series。

## 3. 辅助函数

### 3.1 _resolve_target_indicator_series()

```python
def _resolve_target_indicator_series(condition: dict, df: pd.DataFrame | None) -> pd.Series | None
```

从 condition 中提取 `target_indicator` 和 `target_params`，构造一个 `_SimpleGene` 对象，
通过 `_get_indicator_column()` 在 df 中查找对应的指标列。

查找失败时返回 None。

### 3.2 _evaluate_inner_condition()

```python
def _evaluate_inner_condition(
    indicator_series: pd.Series,
    close_series: pd.Series,
    inner: dict,
    df: pd.DataFrame | None,
) -> pd.Series
```

评估 lookback 条件中的内嵌条件。支持:
- lt, gt, le, ge: 直接比较
- price_above, price_below: 价格比较
- 其他复杂条件: 委托给 `evaluate_condition()`

### 3.3 _SimpleGene

```python
class _SimpleGene:
    __slots__ = ("indicator", "params", "field_name")
```

最小化的 gene-like 对象，仅用于 `_get_indicator_column()` 的参数传递。
避免为查找列构造完整的 SignalGene。

## 4. 信号组合: combine_signals()

```python
def combine_signals(signal_list: list[pd.Series], logic: str) -> pd.Series
```

将多个布尔 Series 按逻辑运算符组合。

**AND 逻辑**:
```python
result = signal_list[0].copy()
for s in signal_list[1:]:
    result = result & s
```
所有信号必须同时为 True。

**OR 逻辑**:
```python
result = signal_list[0].copy()
for s in signal_list[1:]:
    result = result | s
```
任一信号为 True 即可。

**空列表**: 返回全 False 的 Series。
**未知逻辑**: 返回第一个信号。

## 6. 层级评估: evaluate_layer()

```python
def evaluate_layer(layer, df: pd.DataFrame) -> tuple[pd.Series, pd.Series]
```

评估一个 TimeframeLayer 在给定 DataFrame 上的信号。

### 6.1 处理流程

```
1. 遍历 layer.signal_genes
2. 对每个 gene:
   a. _get_indicator_column(df, gene) -> indicator_col
   b. evaluate_condition(indicator_col, close, gene.condition, df=df) -> signal
   c. 根据 gene.role 分类到 4 个列表
3. 合并 entry_triggers + entry_guards -> all_entry
4. combine_signals(all_entry, layer.logic_genes.entry_logic) -> entries
5. 合并 exit_triggers + exit_guards -> all_exit
6. combine_signals(all_exit, layer.logic_genes.exit_logic) -> exits
7. return (entries, exits)
```

**错误处理**: 若某个基因的指标列查找失败 (ValueError)，跳过该基因继续处理。

**角色分类**:

| 角色 | 入场信号 | 出场信号 |
|------|---------|---------|
| ENTRY_TRIGGER | entry_triggers | - |
| ENTRY_GUARD | entry_guards | - |
| EXIT_TRIGGER | - | exit_triggers |
| EXIT_GUARD | - | exit_guards |

注意: trigger 和 guard 在同一侧 (entry/exit) 中以相同权重参与逻辑组合。

## 7. 信号重采样: resample_signals()

```python
def resample_signals(
    high_tf_signals: pd.Series,
    target_index: pd.DatetimeIndex,
) -> pd.Series
```

将高时间框架的信号对齐到执行时间框架的索引。

### 7.1 实现机制

```python
reindexed = high_tf_signals.reindex(target_index, method="ffill")
return reindexed.fillna(False).astype(bool)
```

- 使用 `reindex(method="ffill")` 前向填充: 高时间框架产生信号后，
  该信号在其覆盖的所有低时间框架 bar 上保持为 True
- 空值填充为 False
- 最终转换为布尔类型

### 7.2 使用场景

在 MTF 模式中，日线层产生的信号需要对齐到 4 小时执行时间框架。
日线上一根 bar 覆盖 6 根 4 小时 bar，前向填充确保日线信号在其覆盖区间内持续有效。

## 8. 主入口: dna_to_signals()

```python
def dna_to_signals(
    dna: StrategyDNA,
    enhanced_df: pd.DataFrame,
    dfs_by_timeframe: dict | None = None,
) -> tuple[pd.Series, pd.Series]
```

将 StrategyDNA 转换为 entry/exit 布尔 Series。支持两种模式。

### 8.1 MTF 模式

触发条件: `dna.is_mtf == True` 且 `dfs_by_timeframe is not None`

```
对每个 layer:
  1. 从 dfs_by_timeframe[layer.timeframe] 获取该层 DataFrame
  2. evaluate_layer(layer, layer_df) -> (entries, exits)
  3. 若 layer.timeframe != 执行时间框架:
     resample_signals(entries/exits, enhanced_df.index)
  4. 否则:
     reindex 到 enhanced_df.index (fill_value=False)

收集所有层的 layer_entries / layer_exits

combined_entries = combine_signals(layer_entries, dna.cross_layer_logic)
combined_exits = combine_signals(layer_exits, dna.cross_layer_logic)

去冲突: entries 和 exits 同时为 True 的 bar，仅保留 exit
both = combined_entries & combined_exits
combined_entries = combined_entries & ~both

return (combined_entries, combined_exits)
```

**关键细节**:
- 若某个层没有对应的 DataFrame (dfs_by_timeframe 中缺失)，跳过该层
- 若所有层都被跳过，返回两个全 False 的 Series
- 跨层组合使用 `dna.cross_layer_logic` (AND/OR)
- 去冲突: 同一 bar 不能同时入场和出场，出场优先

### 8.2 单时间框架模式

触发条件: 不满足 MTF 模式条件 (默认路径)

```
1. 遍历 dna.signal_genes
2. 对每个 gene:
   a. _get_indicator_column(enhanced_df, gene) -> indicator_col
   b. evaluate_condition(indicator_col, close, gene.condition, df=enhanced_df) -> signal
   c. 根据 gene.role 分类到 4 个列表

3. all_entry = entry_triggers + entry_guards
   entries = combine_signals(all_entry, dna.logic_genes.entry_logic)
   (空列表返回全 False)

4. all_exit = exit_triggers + exit_guards
   exits = combine_signals(all_exit, dna.logic_genes.exit_logic)
   (空列表返回全 False)

5. 去冲突:
   both = entries & exits
   entries = entries & ~both

return (entries, exits)
```

### 8.3 去冲突机制

两种模式都包含相同的去冲突逻辑:

```python
both = entries & exits
entries = entries & ~both
```

当 entry 和 exit 信号在同一 bar 同时触发时:
- 保留 exit 信号
- 取消 entry 信号

这避免了在同一 bar 上同时开仓和平仓的矛盾情况。

## 9. 数据流全景

### 9.1 单时间框架策略执行路径

```
OHLCV DataFrame
    |
    v
compute_all_indicators()  -- 批量预计算 30 种指标
    |
    v
Enhanced DataFrame (含 100+ 指标列)
    |
    v
dna_to_signals(dna, enhanced_df)
    |
    +-- 对每个 signal_gene:
    |   _get_indicator_column() -> 指标 Series
    |   evaluate_condition() -> 布尔 Series
    |
    +-- 按角色分组 (entry_trigger/guard, exit_trigger/guard)
    |
    +-- combine_signals(all_entry, entry_logic) -> entries
    +-- combine_signals(all_exit, exit_logic) -> exits
    |
    +-- 去冲突 -> (entries, exits)
    |
    v
BacktestEngine.run(dna, enhanced_df)
    |
    v
BacktestResult (equity_curve, trades, metrics)
```

### 9.2 多时间框架策略执行路径

```
多个 OHLCV DataFrame (1d, 4h, ...)
    |
    v
各时间框架独立 compute_all_indicators()
    |
    v
dfs_by_timeframe = {"1d": df_1d, "4h": df_4h, ...}
    |
    v
dna_to_signals(dna, enhanced_df, dfs_by_timeframe)
    |
    +-- 对每个 layer:
    |   evaluate_layer(layer, dfs_by_timeframe[layer.timeframe])
    |       -> (layer_entries, layer_exits)
    |
    +-- 非执行时间框架的信号:
    |   resample_signals() 前向填充到执行时间框架
    |
    +-- combine_signals(所有层entries, cross_layer_logic) -> combined_entries
    +-- combine_signals(所有层exits, cross_layer_logic) -> combined_exits
    |
    +-- 去冲突 -> (entries, exits)
    |
    v
BacktestEngine.run(dna, enhanced_df, dfs_by_timeframe)
    |
    v
BacktestResult
```

## 10. 错误处理与容错

整个信号生成过程采用"最佳努力"策略:

1. **指标列不存在**: `_get_indicator_column()` 抛出 ValueError，`evaluate_layer()` 和
   `dna_to_signals()` 捕获后跳过该基因，继续处理其他基因。

2. **条件评估失败**: `evaluate_condition()` 对未知条件类型返回全 False。

3. **MTF 数据缺失**: 若 dfs_by_timeframe 中缺少某个时间框架的数据，跳过该层。

4. **空信号列表**: 若某侧 (entry/exit) 无任何信号基因，返回全 False 的 Series。

5. **NaN 处理**: 每个信号基因评估后立即 `fillna(False)`，确保布尔运算的确定性。

这种设计确保即使 DNA 中包含无效的指标/参数组合，也不会导致整个信号生成失败，
只是该基因不产生信号。这在进化搜索中至关重要，因为变异可能产生大量无效组合。
