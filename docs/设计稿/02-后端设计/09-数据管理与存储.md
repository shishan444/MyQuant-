# 09 - 数据管理与存储

## 1. 概述

数据管理模块负责市场数据的获取、导入、存储和分发。系统支持两种数据源 (Binance API 和 CSV 导入),使用 Parquet 格式进行高效的列式存储,并提供多时间框架数据加载能力。所有数据操作均围绕 OHLCV (开高低收量) 标准格式展开。

**核心源文件**:
- `core/data/fetcher.py` - Binance API 数据获取
- `core/data/csv_importer.py` - CSV 文件导入
- `core/data/storage.py` - Parquet 存储层
- `core/data/updater.py` - 增量数据更新
- `core/data/mtf_loader.py` - 多时间框架数据加载
- `api/routes/data.py` - 数据管理 API 路由

## 2. 数据源

### 2.1 Binance API (fetcher.py)

#### fetch_klines() 函数

```python
def fetch_klines(
    symbol: str = "BTCUSDT",
    interval: str = "4h",
    start_str: str = "2 year ago UTC",
    end_str: str | None = None,
    api_key: str = "",
    api_secret: str = "",
) -> pd.DataFrame:
```

**流程**:
1. 创建 Binance `Client` 实例 (公开端点不需要 API Key)
2. 调用 `get_historical_klines()` 获取历史K线
3. 构造 DataFrame,原始 12 列映射为列名:
   - `timestamp, open, high, low, close, volume, close_time, quote_volume, trades, taker_buy_base, taker_buy_quote, ignore`
4. 保留必要列: `timestamp, open, high, low, close, volume, trades`
5. 类型转换: OHLCV 转为 float, trades 转为 int
6. 时间戳转为 UTC DatetimeIndex (毫秒级)
7. 去重: 保留首次出现的索引

**返回**: DataFrame with DatetimeIndex(UTC), columns = [open, high, low, close, volume, trades]

### 2.2 CSV 导入 (csv_importer.py)

#### 支持的 CSV 格式

**Binance 官方格式**:
- 无表头行
- 12 列: `open_time, open, high, low, close, volume, close_time, quote_volume, trades, taker_buy_base, taker_buy_quote, ignore`
- 第一列为毫秒/微秒时间戳

**通用 OHLCV 格式**:
- 有表头行
- 列名通过映射表识别:

| 原始列名 | 映射目标 |
|----------|----------|
| open_time, timestamp, date, datetime | timestamp |
| open | open |
| high | high |
| low | low |
| close | close |
| volume | volume |

#### 格式自动检测 (detect_format)

```python
def detect_format(path: Path) -> ImportFormat:
```

检测逻辑:
1. 读取文件首行
2. 若首字段是可识别的列名 (open_time, timestamp, date, datetime, open, high, low, close, volume): 判定为 `GENERIC_OHLCV`
3. 若有 12 列且第二字段为数值: 判定为 `BINANCE_OFFICIAL`
4. 若首字段为整数且有 12 列: 判定为 `BINANCE_OFFICIAL`
5. 默认: `GENERIC_OHLCV`


| 数字位数 | 精度 |
|----------|------|
| <= 13 | 毫秒 (ms) |

#### 文件名解析 (parse_filename)


**解析规则**: 从文件名提取交易对和时间框架。

若文件名不匹配模式,返回 `(None, None)`,需调用方显式指定。

### validate_ohlcv()

```python
def validate_ohlcv(df: pd.DataFrame) -> list[str]:
```

**校验项**:

| 校验 | 规则 | 错误示例 |
|------|------|----------|
| NaN 检查 | OHLCV 各列不允许有 NaN | "Column 'close' has 3 NaN values" |
| High 一致性 | high >= max(open, close, low) | "Row 2024-01-01: high (42000) < max(O,C,L) (42100)" |
| Low 一致性 | low <= min(open, close, high) | "Row 2024-01-01: low (41500) > min(O,C,H) (41400)" |
| Volume 非负 | volume >= 0 | "Column 'volume' has 2 negative values" |

**High/Low 校验使用向量化操作**,仅报告首个违规行。

## 4. Parquet 存储层 (storage.py)

### 4.1 存储引擎

使用 PyArrow 引擎进行 Parquet 读写:

```python
df.to_parquet(path, engine="pyarrow")
pd.read_parquet(path, engine="pyarrow")
```

### 4.2 核心函数

**save_parquet(df, path)**:
- 自动创建父目录
- 使用 PyArrow 引擎写入
- 覆盖写入模式

**load_parquet(path)**:
- 使用 PyArrow 引擎读取
- 返回 DataFrame with DatetimeIndex

**get_latest_timestamp(path)**:
- 文件不存在或为空返回 None
- 返回索引最大值 (最新时间戳)
- 用于增量更新的起点判断

**merge_parquet(new_df, path)**:
- 若文件不存在,等同于 save_parquet
- 加载已有数据,与新数据 concat
- 按索引去重: `keep="last"` (新数据优先)
- 按索引排序后写回

### 4.3 文件命名规范

```
{SAFE_SYMBOL}_{TIMEFRAME}.parquet

示例:
  BTCUSDT_4h.parquet
  ETHUSDT_1d.parquet
```

其中 `SAFE_SYMBOL` 是移除所有非字母数字字符后的交易对名称。

## 5. 增量更新 (updater.py)

### update_market_data()

```python
def update_market_data(
    symbol: str = "BTCUSDT",
    interval: str = "4h",
    data_dir: Path | None = None,
    history_years: int = 2,
    api_key: str = "",
    api_secret: str = "",
) -> pd.DataFrame:
```

**流程**:

```
update_market_data()
    |
    +-- get_latest_timestamp(path)
    |       |
    |       +-- None (文件不存在)
    |       |       +-- 全量获取: fetch_klines(start="{N} year ago UTC")
    |       |       +-- save_parquet(df, path)
    |       |
    |       +-- 有值 (文件已存在)
    |               +-- 增量获取: fetch_klines(start=latest_ts)
    |               +-- merge_parquet(new_df, path)
    |
    +-- 返回完整的 DataFrame
```

**增量合并**: 新获取的数据与已有数据合并,去重时保留最新值 (`keep="last"`),排序后写回。

## 6. CSV 导入流程

### 6.1 单文件导入 (import_csv)

```python
def import_csv(
    path: Path,
    data_dir: Path,
    symbol: str | None = None,
    interval: str | None = None,
    mode: ImportMode = ImportMode.MERGE,
) -> CsvImportResult:
```

**流程**:
1. 从文件名自动检测 symbol/interval (可手动覆盖)
2. 读取 CSV 并自动检测格式
3. 执行 OHLCV 数据校验
4. 根据 mode 写入 Parquet:
   - `MERGE`: 调用 `merge_parquet()` 合并
   - `REPLACE`: 调用 `save_parquet()` 覆盖
   - `NEW`: 调用 `save_parquet()` 新建
   - 文件不存在时,所有模式等同于新建

### 6.2 批量导入 (import_csv_batch)

```python
def import_csv_batch(
    paths: Sequence[Path],
    data_dir: Path,
    symbol: str | None = None,
    interval: str | None = None,
    mode: ImportMode = ImportMode.MERGE,
) -> CsvImportResult:
```

**流程**:
1. 逐文件读取 CSV 并校验
2. 从首个文件检测 symbol/interval 和格式信息
3. 合并所有文件: `pd.concat(all_dfs)`
4. 全局去重: `keep="first"`,按索引排序
5. 写入 Parquet (与单文件相同的 mode 逻辑)

### 6.3 CsvImportResult

## 7. 多时间框架数据加载 (mtf_loader.py)

### 7.1 时间框架别名

```python
_TF_ALIASES = {
    "1h": ["1h", "60m"],
    "4h": ["4h"],
    "1d": ["1d", "1D"],
    "15m": ["15m"],
    "5m": ["5m"],
    "1m": ["1m"],
    "30m": ["30m"],
    "3d": ["3d"],
}
```

### 7.2 find_parquet()

```python
def find_parquet(data_dir: Path, safe_symbol: str, timeframe: str) -> Optional[Path]:
```

查找逻辑:
1. 首先尝试精确匹配: `{symbol}_{timeframe}.parquet`
2. 然后尝试别名: 遍历 `_TF_ALIASES[timeframe]` 中的每个别名
3. 未找到返回 None

**应用场景**: 文件名可能使用 `60m` 而非 `1h`,或 `1D` 而非 `1d`。

### 7.3 load_and_prepare_df()

```python
def load_and_prepare_df(
    data_dir: Path,
    symbol: str,
    timeframe: str,
    data_start: Optional[str] = None,
    data_end: Optional[str] = None,
    min_bars: int = 50,
) -> Optional[pd.DataFrame]:
```

**流程**:
1. 清理 symbol (移除非字母数字字符)
2. `find_parquet()` 查找文件
3. `load_parquet()` 加载数据
4. 检查最少K线数 (默认 50)
5. 按日期范围切片
6. 再次检查最少K线数
7. 调用 `compute_all_indicators()` 计算指标
8. 返回增强后的 DataFrame (OHLCV + 指标列)

任何步骤失败均返回 None。

### 7.4 load_mtf_data()

```python
def load_mtf_data(
    data_dir: Path,
    symbol: str,
    exec_timeframe: str,
    enhanced_df: pd.DataFrame,
    needed_tfs: Set[str],
    data_start: Optional[str] = None,
    data_end: Optional[str] = None,
) -> Optional[Dict[str, pd.DataFrame]]:
```

**流程**:
1. 以执行时间框架的 DataFrame 为基础
2. 遍历需要的额外时间框架:
   - `find_parquet()` 查找文件
   - 加载并按日期范围过滤
   - 检查最少 50 根K线
   - 计算指标
3. 返回 `{timeframe: DataFrame}` 字典 (包含执行时间框架)
4. 若无法加载任何额外时间框架,返回 None

## 11. 设计约束与注意事项

1. **Parquet 列式存储**: 使用 PyArrow 引擎,适合分析型查询,不支持行级更新
2. **时间戳 UTC**: 所有数据的索引均为 UTC 时区的 DatetimeIndex
3. **安全符号处理**: 交易对名称中的非字母数字字符在文件操作前被移除
4. **增量更新不可逆**: `merge_parquet` 使用 `keep="last"`,新数据总是覆盖旧数据中的相同时间点
5. **最少数据量**: `load_and_prepare_df` 和 `load_mtf_data` 均要求至少 50 根K线
6. **指标延迟计算**: 指标不在存储时计算,而是在加载时按需计算 (`compute_all_indicators`)
7. **临时文件清理**: CSV 导入使用临时文件,无论成功与否都在 finally 块中清理
8. **API 数据库**: 数据集元信息 (不含OHLCV数据本身) 存储在 SQLite 中,通过 `db_ext.py` 管理
9. **文件名依赖**: 多个功能依赖文件名格式 `{SYMBOL}_{TIMEFRAME}.parquet`,非标准命名将导致查找失败
10. **别名查找有限**: 时间框架别名仅覆盖常见格式 (`1h/60m`, `1d/1D`),其他变体不会被识别
