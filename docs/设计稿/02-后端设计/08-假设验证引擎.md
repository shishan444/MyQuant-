# 08 - 假设验证引擎

## 1. 概述

假设验证引擎提供 WHEN/THEN 模式的历史假设验证能力。用户定义一组 WHEN 条件 (触发条件) 和 THEN 条件 (预期结果),引擎在历史数据上搜索所有触发点,并统计 THEN 条件的达成率、分布、百分位数等统计信息。引擎支持 15+ 种动作类型、多时间框架条件和复杂的模式检测。

**核心源文件**:
- `core/validation/engine.py` - 验证引擎主体
- `core/validation/patterns.py` - 模式检测算法
- `core/validation/mtf.py` - 多时间框架数据加载与合并

---

## 2. 整体架构

```
validate_hypothesis(pair, timeframe, start, end, when_conditions, then_conditions)
    |
    +-- 1. 加载 OHLCV 数据 (load_parquet)
    +-- 2. 计算指标 (compute_all_indicators)
    +-- 3. MTF 数据加载与合并 (可选)
    |       +-- load_mtf_data()
    |       +-- merge_to_base()
    +-- 4. 评估 WHEN 条件 (_evaluate_conditions)
    |       +-- _evaluate_single_condition() x N
    |       +-- AND/OR 逻辑组合
    +-- 5. 查找触发点 (trigger_indices)
    +-- 6. 逐触发点验证 THEN 条件
    |       +-- 前向窗口切取
    |       +-- _check_then_conditions()
    |       +-- 记录 TriggerRecord
    +-- 7. 统计分析
    |       +-- 匹配率
    |       +-- _build_distribution()
    |       +-- 百分位数
    |       +-- 集中度
    |       +-- 信号频率
    |       +-- 极值
    +-- 8. 返回 ValidationResult
```

## 6. 目标解析 (_resolve_target)

`_resolve_target(df, target)` 的解析顺序:

1. **数值类型**: 若 target 已是 int/float,直接返回
2. **数值字符串**: 尝试 `float(target)` 解析
3. **跨时间框架格式**: `"4h:ema_20"` -> 查找 `"ema_20_4h"` 列
4. **精确列名**: 在 DataFrame 列中精确匹配
5. **模糊匹配**: 去除下划线后进行子串匹配

---

## 7. 对象解析 (_resolve_subject)

`_resolve_subject(df, subject)` 支持以下映射:

| subject | 解析结果 |
|---------|----------|
| `close`, `price` | `df["close"]` |
| `open` | `df["open"]` |
| `high` | `df["high"]` |
| `low` | `df["low"]` |
| `volume` | `df["volume"]` |
| `kdj` | `df["stoch_k"]` |
| 其他 | 模糊匹配列名 |

模糊匹配逻辑: 将 subject 和列名都去除下划线并转小写后,进行子串匹配。

## 8. 模式检测算法 (patterns.py)

### 8.1 顶背离 (detect_divergence_top)

**定义**: 价格创新高,但指标未创新高。

**算法**:
1. 在 lookback (默认 20) 根K线的滑动窗口内查找价格的局部极大值
2. 取最近两个极大值点
3. 若价格的第二高点 > 第一高点,且指标的第二高点 < 第一高点,标记为顶背离

**局部极大值检测** (`_find_local_maxima`):
- 严格大于左右邻居的点
- 端点若大于唯一邻居也纳入

### 8.2 底背离 (detect_divergence_bottom)

**定义**: 价格创新低,但指标未创新低。

**算法**: 与顶背离对称,查找价格的局部极小值,比较最近两个极小值点。

### 8.3 连续上涨 (detect_consecutive_up)

**定义**: 指标值连续 count (默认 3) 根K线严格递增。

**算法**: 对每根K线,检查前 count 根到当前K线是否严格递增 (排除 NaN)。

### 8.4 连续下跌 (detect_consecutive_down)

与连续上涨对称,检查严格递减。

### 8.5 触碰反弹 (detect_touch_bounce)

**定义**: 价格触碰指标线后反弹。

**参数**:
- `direction`: "support" (支撑) 或 "resistance" (阻力)
- `proximity_pct`: 接近阈值 (默认 1%)
- `bounce_pct`: 最小反弹幅度 (默认 0.5%)

**支撑模式**:
1. Low 值接近指标线: `abs(low - line) <= tolerance`
2. Close 在指标线之上: `close > line`
3. 下一根K线反弹: `close_next > close * (1 + bounce_pct)`

**阻力模式**: 对称逻辑,使用 High 值和向下反弹。

### 8.6 角色互换 (detect_role_reversal)

**定义**: 支撑位变为阻力位,或阻力位变为支撑位。

**参数**:
- `role`: "resistance" (支撑变阻力) 或 "support" (阻力变支撑)
- `lookback`: 回溯K线数 (默认 10)

**阻力模式** (支撑变阻力):
1. lookback 根前: 价格在指标线之上 (`close > line`)
2. 当前: 价格在指标线之下 (`close < line`)

**支撑模式**: 条件相反。

## 9. THEN 条件验证

### 9.1 _check_then_conditions()

```python
def _check_then_conditions(
    window_df: pd.DataFrame,      # 前向窗口数据
    conditions: List[Dict],       # THEN 条件列表
    trigger_price: float,         # 触发价格
    trigger_loc: int = 0,         # 触发位置
    max_window: int = 8,          # 最大窗口
) -> bool:
```

### 9.2 THEN 动作类型

**价格下跌类** (`drop`, `lt`, `le`):
```
min_price = window_df["close"].min()
change = ((min_price - trigger_price) / trigger_price) * 100
匹配: change <= threshold (drop), change < threshold (lt), change <= threshold (le)
```

**价格上涨类** (`rise`, `gt`, `ge`):
```
max_price = window_df["close"].max()
change = ((max_price - trigger_price) / trigger_price) * 100
匹配: change >= threshold (rise), change > threshold (gt), change >= threshold (ge)
```

### 9.3 独立窗口

每个 THEN 条件可以有独立的 `window` 字段:
```python
cond_window = cond.get("window")
if cond_window is not None and cond_window < max_window:
    cond_df = window_df.iloc[:cond_window]
else:
    cond_df = window_df
```

允许不同 THEN 条件在不同时间跨度内验证。

### 9.4 匹配逻辑

任一 THEN 条件满足即返回 True (OR 逻辑)。

## 12. 分布构建算法

### 12.1 _build_distribution()

```python
def _build_distribution(
    changes: List[float],
    matches: List[bool],
) -> List[DistributionBucket]:
```

**步骤**:
1. 计算所有变化百分比的 min 和 max
2. 桶数量: `min(10, max(5, len(changes) // 5))`
3. 桶宽度: `(max - min) / num_buckets`
4. 将每个变化值分配到对应桶
5. 统计每个桶内的匹配和不匹配计数
6. 最后一个桶使用 `<=` 闭合

## 13. 辅助条件函数

### 13.1 _touch_condition

```python
tolerance = abs(target) * 0.005
return abs(series - target) <= max(tolerance, series.std() * 0.1)
```

容差取 0.5% 目标值与 10% 标准差中的较大值。

### 13.2 _spike_condition / _shrink_condition

```python
rolling_avg = series.rolling(window=20, min_periods=1).mean()
# spike: series >= rolling_avg * multiplier
# shrink: series <= rolling_avg * multiplier
```

默认乘数: spike 为 2.0, shrink 为 0.5。

### 13.3 _cross_above_series_condition

```python
(subject.shift(1) < target_series.shift(1)) & (subject >= target_series)
```

前一周期在下方,当前周期到达或超过。

### 13.4 _lookback_condition

在回溯窗口内检查内部条件是否满足:
- `mode="any"`: 窗口内任一K线满足即可
- 使用 `rolling().apply(any)` 实现

## 14. 设计约束与注意事项

1. **THEN 逻辑为 OR**: 多个 THEN 条件之间是 OR 关系,任一满足即视为匹配
2. **触发价格基准**: THEN 的变化百分比基于触发时刻的 close 价格计算
3. **无 THEN 默认逻辑**: 若所有 THEN 条件的 action 都不在预定义列表中,尝试用第一个条件做默认 drop/rise 判断
4. **MTF 合并防偷看**: 高时间框架数据的时间戳向前偏移一个基准单位,确保不会使用未来数据
5. **分布桶自适应**: 桶数量根据样本数动态调整 (最少 5 个,最多 10 个)
6. **极值仅记录最差**: 当前只记录 change_pct 最小的触发事件
7. **警告信息收集**: 解析失败的条件 (未识别的 subject/action/target) 会记录到 warnings 列表,但不阻断流程
8. **安全符号处理**: 交易对中的非字母数字字符被移除后用于文件名查找
