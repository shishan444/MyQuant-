# 06 - 回测引擎设计

## 1. 概述

回测引擎负责将策略 DNA 转化为具体的交易信号,通过 vectorbt 框架模拟历史交易,并考虑杠杆资金费率和爆仓风险,最终输出结构化的回测结果。引擎同时集成了 Walk-Forward 验证器,用于评估策略的样本外表现,防止过拟合。

**核心源文件**:
- `core/backtest/engine.py` - 回测引擎与 BacktestResult
- `core/backtest/walk_forward.py` - Walk-Forward 验证器

## 3. _build_portfolio() 构建投资组合

### 3.1 信号生成

调用 `dna_to_signals(dna, enhanced_df, dfs_by_timeframe)` 将策略 DNA 转化为布尔型的入场和出场信号序列。该方法支持:
- 单时间框架策略: 直接在 `enhanced_df` 上计算
- 多时间框架策略: 结合 `dfs_by_timeframe` 中各时间框架的数据

### 3.2 Portfolio 参数构建

```python
portfolio_kwargs = {
    "close": close,                          # 收盘价序列
    "entries": entries,                      # 入场信号 (布尔Series)
    "exits": exits,                          # 出场信号 (布尔Series)
    "open": open_,                           # 开盘价 (用于滑点模拟)
    "high": high,                            # 最高价 (用于止损模拟)
    "low": low,                              # 最低价 (用于止损模拟)
    "init_cash": self.init_cash,             # 初始资金
    "fees": self.fee,                        # 手续费
    "slippage": self.slippage,               # 滑点
    "sl_stop": dna.risk_genes.stop_loss,     # 止损比例
    "tp_stop": dna.risk_genes.take_profit,   # 止盈比例 (可None)
    "size": position_size * leverage,         # 仓位大小
    "size_type": "percent",                  # 百分比模式
    "accumulate": False,                     # 不累积加仓
    "direction": direction_map[direction],    # 0=long, 1=short
}
```

### 3.3 方向映射

```python
direction_map = {"long": 0, "short": 1}
```

### 3.4 仓位计算

```
size = dna.risk_genes.position_size * dna.risk_genes.leverage
```

- `position_size`: 基础仓位比例 (如 0.3 表示 30%)
- `leverage`: 杠杆倍数
- 实际仓位 = 资金 * position_size * leverage

## 5. 资金费率模型

### 5.1 _apply_funding_costs()

针对杠杆仓位的资金费率扣减:

```python
def _apply_funding_costs(
    equity_curve: pd.Series,
    leverage: int,
    timeframe: str,
) -> tuple[pd.Series, float]:
```

**核心参数**:
- `RATE_PER_8H = 0.001`: 每 8 小时费率 0.1%
- 仅在 `leverage > 1` 时生效

### 5.2 计算逻辑

1. 根据时间框架计算每根K线对应的小时数:
   - 1m: 1/60h, 5m: 5/60h, 15m: 0.25h, 30m: 0.5h
   - 1h: 1h, 4h: 4h, 1d: 24h, 3d: 72h

2. 计算每根K线覆盖的 8 小时周期数:
   ```
   periods_per_bar = ceil(hours_per_bar / 8)
   ```

3. 计算借款比例和费率:
   ```
   borrowed_ratio = (leverage - 1) / leverage
   cost_rate = RATE_PER_8H * periods_per_bar * borrowed_ratio
   ```

4. 逐K线从权益中扣除:
   ```
   cost = previous_equity * cost_rate
   adjusted_equity[i] = raw_equity[i] - cost
   ```

### 5.3 费率计算示例

以 4h 时间框架、3 倍杠杆为例:
```
hours_per_bar = 4
periods_per_bar = ceil(4/8) = 1
borrowed_ratio = (3-1)/3 = 0.667
cost_rate = 0.001 * 1 * 0.667 = 0.000667
每根K线扣减 = 前一根权益 * 0.000667
```

## 6. 爆仓检查

### 6.1 _check_liquidation()

```python
def _check_liquidation(
    equity_curve: pd.Series,
    leverage: int,
    init_cash: float,
) -> bool:
```

**仅在 `leverage > 1` 时生效**。

### 6.2 爆仓条件

```
maintenance = init_cash * (1 - 0.9 / leverage)
```

- 当权益低于 `maintenance` 时触发爆仓
- 爆仓意味着保证金损失超过 90% 的有效资本

### 6.3 爆仓处理

- 找到首个爆仓K线的索引
- 将该索引及之后的所有权益值设为 0
- 返回 `True` 标记已爆仓

### 6.4 示例

以 `init_cash = 100000`, `leverage = 3` 为例:
```
maintenance = 100000 * (1 - 0.9/3) = 100000 * 0.7 = 70000
当权益 < 70000 时触发爆仓
```

```python
total_trades_val = portfolio.trades.count()
```

```python
if total_trades > 0:
    trade_pnl = portfolio.trades.pnl        # 每笔交易盈亏数组
    pnl_arr = np.array(trade_pnl).flatten()
    winning = (pnl_arr > 0).sum()
```

`trade_win_rate` 和 `trade_returns` 在后续的评分计算中优先于 K 线级别的统计数据。

## 9. Walk-Forward 验证

### 9.1 WalkForwardValidator 类

```python
class WalkForwardValidator:
    def __init__(
        self,
        train_months: int = 3,          # 训练窗口 (月)
        slide_months: int = 1,          # 滑动步长 (月)
        train_weight: float = 0.4,      # 训练权重
        val_weight: float = 0.6,        # 验证权重
        template_name: str = "profit_first",
    ):
```

### 9.2 validate() 方法

```python
def validate(
    self,
    dna: StrategyDNA,
    enhanced_df: pd.DataFrame,
    val_months: Optional[List[str]] = None,  # 共享验证月份
) -> Dict:
    # 返回: { wf_score, n_rounds, rounds }
```

### 9.3 Walk-Forward 流程

```
validate(dna, enhanced_df, val_months)
    |
    +-- 1. 生成月份边界 (pd.date_range(freq="MS"))
    +-- 2. 按滑动步长遍历训练窗口
    |       |
    |       +-- train_start = month_starts[i]
    |       +-- train_end = month_starts[i + train_months]
    |       +-- val_month = 随机选择训练窗口外的月份
    |       |
    |       +-- 切片训练数据 (train_mask)
    |       +-- 切片验证数据 (val_mask)
    |       |
    |       +-- engine.run(dna, train_df)   训练集回测
    |       +-- engine.run(dna, val_df)     验证集回测
    |       |
    |       +-- score_strategy(train_metrics)   训练分数
    |       +-- score_strategy(val_metrics)     验证分数
    |       +-- combined = train * train_weight + val * val_weight
    |
    +-- 3. wf_score = mean(combined_scores)
```

### 9.4 训练窗口滑动

```
数据范围: |------2024-01------2024-12------|
窗口1:    [train: 1-3月]  [val: 随机非1-3月]
窗口2:    [train: 2-4月]  [val: 随机非2-4月]
窗口3:    [train: 3-5月]  [val: 随机非3-5月]
...
```

- 训练窗口: `train_months` 个月 (默认 3)
- 滑动步长: `slide_months` 个月 (默认 1)
- 验证月份: 从训练窗口外的所有月份中随机选择

### 9.5 共享验证月份

`val_months` 参数允许为同代策略共享相同的验证月份,确保公平比较:
- 若指定,按索引匹配预生成的验证月份
- 若无法匹配 (年份和月份),回退到随机选择

### 9.6 数据量检查

每轮训练和验证都检查数据量:
- 训练集最少 20 根 K 线
- 验证集最少 5 根 K 线
- 不满足则跳过该轮

### 9.7 综合分数计算

```
combined_score = train_score * train_weight + val_score * val_weight
wf_score = mean(combined_scores for all rounds)
```

默认权重: 训练 40%, 验证 60%。更侧重样本外表现。

### 9.8 返回结果

```python
{
    "wf_score": float,           # Walk-Forward 综合分数
    "n_rounds": int,             # 有效轮次数
    "rounds": [                  # 每轮详情
        {
            "round_num": int,
            "train_start": str,   # 训练开始日期
            "train_end": str,     # 训练结束日期
            "val_month": str,     # 验证月份
            "train_score": float,
            "val_score": float,
            "combined_score": float,
        },
        ...
    ]
}
```

## 12. 设计约束与注意事项

1. **资金费率顺序**: 先扣除资金费率,再检查爆仓。权益曲线中已包含费率扣减的影响
2. **爆仓不可逆**: 一旦触发爆仓,该时间点后的所有权益被置零,后续无法恢复
3. **vectorbt 依赖**: Portfolio 使用 `from_signals()` 方法,传入 OHLCV 数据用于滑点和止损的精确模拟
4. **无杠杆不计费**: `leverage <= 1` 时跳过资金费率和爆仓检查
5. **Walk-Forward 独立引擎**: `WalkForwardValidator` 内部创建新的 `BacktestEngine()` 实例,使用默认参数 (init_cash=100000)
6. **最少数据要求**: Walk-Forward 要求数据范围至少覆盖 `train_months + 1` 个月份边界点
7. **权益曲线类型处理**: 若 `portfolio.value()` 返回 DataFrame (多列),取第一列
