## 1. 页面全景

MyQuant 共包含 6 个一级页面，通过左侧导航栏切换。默认落地页为策略实验室（`/lab`）。

```
+------------------------------------------------------------------+
|  [Logo] MyQuant                                                   |
+----------+-------------------------------------------------------+
|          |  Header (动态页面标题)                                   |
| 策略实验室 +-------------------------------------------------------+
| 进化中心  |                                                       |
| 策略库    |              页面内容区域                                |
| 模拟交易  |                                                       |
| 数据管理  |                                                       |
| 设置      |                                                       |
|          |                                                       |
| [折叠/展开]|                                                       |
| v1.0.0   |                                                       |
+----------+-------------------------------------------------------+
```

Sidebar 支持折叠/展开切换。折叠后宽度缩至 w-16，仅显示图标，鼠标悬停显示 Tooltip。
展开时宽度 w-60，显示图标和文字标签。

### 页面清单

| 序号 | 页面名称 | 路由 | 导航图标       | 核心定位                           |
|------|---------|------|---------------|-----------------------------------|
| 1    | 策略实验室 | /lab | FlaskConical  | 假设验证 + 回测验证，默认落地页      |
| 2    | 进化中心  | /evolution | Dna     | 遗传算法策略进化与发现               |
| 3    | 策略库    | /strategies | BookOpen | 保存的策略的管理与查看             |
| 4    | 模拟交易  | /trading | TrendingUp  | 模拟交易运行面板（占位）            |
| 5    | 数据管理  | /data | Database       | 行情数据导入与管理                  |
| 6    | 设置     | /settings | Settings    | 全局配置与参数默认值                |

**路由映射**（源自 `App.tsx`）：

```
/ (index) --> Navigate to /lab (redirect)
/lab       --> Lab 组件
/evolution --> Evolution 组件
/strategies--> Strategies 组件
/trading   --> Trading 组件
/data      --> DataManagement 组件
/settings  --> Settings 组件
```

全局使用 `QueryClientProvider`（TanStack Query），默认 staleTime 30 秒，重试 1 次。

## 2. 完整功能清单（按模块分组）

### 2.1 策略实验室（`/lab`）

策略实验室是 MyQuant 的核心工作台，提供两种工作模式，通过顶部模式切换器选择。

#### 2.1.1 假设验证模式

**数据源配置：**
- 交易对选择：BTCUSDT / ETHUSDT / BNBUSDT / SOLUSDT，动态从已导入数据集生成选项
- 时间周期选择：15m / 1h / 4h / 1d / 1w，根据选中交易对的可用数据集动态过滤
- 日期范围：手动输入起止日期，支持快捷按钮（近3月 / 近6月 / 近1年 / 今年 / 全部）
- 自动日期范围：切换交易对或周期时，自动适配数据集的时间范围

**WHEN 条件构建（ConditionPillGroup）：**
- 支持的主体（subject）：close、volume、rsi、bb_upper、bb_lower 等指标值
- 支持的动作（action）：touch、breakout、drop、rise、gt、lt 等
- 支持的逻辑连接：AND / OR
- 支持多时间框架条件（跨周期引用），通过 TimeframeSelector 选择参考周期
- 条件以药丸（Pill）形式展示，支持添加/删除/编辑

**THEN 条件构建（ConditionPillGroup）：**
- 交互方式与 WHEN 条件一致
- 额外支持观察窗口（window）参数，指定触发后的观察 K 线数

**预设假设模板（3 个内置示例）：**
- 上轨触碰后下跌：close touch bb_upper --> close drop 0, window 8
- 放量突破后上涨：close breakout bb_upper AND volume spike 2 --> close rise 0, window 8
- RSI 超买后回调：rsi gt 70 --> close drop 0, window 12

**验证与结果展示：**
- 验证结论面板（ValidationConclusion）：符合率、触发总数、匹配/不匹配计数
- K 线图表（KlineChart）：OHLCV 数据 + 触发点标记（颜色区分匹配/不匹配）
- 图表工具栏（ChartToolbar）：缩放、重置、全屏
- 指标叠加：EMA（多条、可配置颜色）、BOLL（布林带）、RSI，参数来自全局设置
- 成交量副图
- 分布图表（DistributionChart）：涨跌幅分布直方图
- 参考面板（ReferencePanel）：百分位数、集中度、信号频率、极值统计
- 触发记录表格（TriggerTable）：逐条展示触发记录，支持定位到图表、查看详情
- 触发详情抽屉（TriggerDetailDrawer）：单条记录的完整指标值快照
- 低符合率提示：符合率低于 20% 时显示优化建议
- 无匹配提示：无触发记录时提供调整条件/扩大时间范围的建议

**保存为策略：**
- 通过 SaveStrategyDialog 填写名称、描述、标签
- 自动从验证条件生成 DNA（通过 `generateDnaFromValidation` 函数）
- 保存到策略库，来源标记为 `lab`

#### 2.1.2 策略回测模式

**入口方式：**
- 从进化中心跳转：通过 React Router `state` 传递 DNA、symbol、timeframe、dataStart、dataEnd
- 手动切换到回测模式：显示空状态提示"请从进化中心选择一个策略进行可视化验证"

**回测执行（BacktestModePanel）：**
- 页面加载时自动执行回测（调用 `POST /api/strategies/backtest`）
- 同时加载 OHLCV K 线数据用于图表展示
- 回测结果展示：
  - 回测指标面板（BacktestMetricsPanel）：总收益率、夏普比率、最大回撤、胜率、交易次数、评分
  - 权益曲线图表（EquityCurveChart）：资金随时间的变化
  - K 线图表 + 交易信号标注：买入/卖出信号以不同颜色标记
  - DNA 详情展示（StrategyDetail）：信号基因、逻辑基因、执行基因、风控基因
- 支持保存策略到策略库

### 2.2 进化中心（`/evolution`）

进化中心基于遗传算法自动从历史数据中进化交易策略。

#### 2.2.1 探索模式选择

通过 SegmentedControl 切换两种模式：
- **自动探索（auto）**：从默认 DNA 出发，遗传算法全自动搜索最优策略
- **种子探索（seed）**：用户提供初始 DNA 作为种子，在其基础上定向进化

#### 2.2.2 自动探索配置（AutoConfigForm）

| 配置项             | 可选值/范围                                              | 默认值       |
|-------------------|--------------------------------------------------------|-------------|
| 交易对（symbol）    | 动态从 available-sources 获取                           | BTCUSDT     |
| 时间周期池          | 1d / 4h / 1h / 30m / 15m / 5m / 1m / 3d               | ["4h"]      |
| 指标池              | 五大类共 30+ 指标（见下方指标列表）                       | ["EMA","RSI"] |
| 评分模板            | 收益优先 / 稳健优先 / 风控优先                           | 收益优先     |
| 种群大小            | 数字输入                                                | 15          |
| 最大进化代数        | 数字输入                                                | 200         |
| 目标分数            | 数字输入                                                | 80          |
| 杠杆倍数            | 1x / 2x / 3x / 5x / 10x                                | 1x          |
| 方向               | 做多 / 做空 / 混合探索                                  | 做多        |
| 数据日期范围        | 起止日期                                                | 可选        |

**多周期池角色分配规则：**
- 周期按持续时间从长到短排序，依次分配角色：趋势 / 判断 / 确认 / 执行
- 执行时间周期取池中最短周期
- 后端自动处理周期排序和别名映射（如 1h 和 60m 等价）

**快速预设（QuickPresets，3 个内置方案）：**
- RSI 超卖反弹：RSI+BB, 4H 单周期, 收益优先
- EMA 趋势跟踪：EMA+ADX, 4H 单周期, 稳健优先
- 多周期趋势共振：1D:EMA + 4H:RSI+BB, 收益优先

#### 2.2.3 种子探索配置（SeedConfigForm）

- 交易对选择
- 初始 DNA 输入（支持从策略库或历史探索结果导入）
- 评分模板、种群大小、最大代数、目标分数、杠杆、方向等参数配置

#### 2.2.4 探索进度监控

- 进度面板（ProgressPanel）：当前代数/最大代数进度条、当前最佳分数/目标分数
- 任务状态：等待中（pending） / 探索中（running） / 已暂停（paused） / 已停止（stopped） / 已完成（completed）
- 操作按钮：暂停 / 恢复 / 停止（停止需确认弹窗 ConfirmDialog）
- 分数趋势图（ScoreTrendChart）：每代最佳分数和平均分数的折线图
- WebSocket 实时通信（`/ws/evolution/{task_id}`）：推送 generation_complete 和 evolution_complete 事件
- 配置折叠：任务运行中自动折叠配置区域，完成后自动展开

#### 2.2.5 发现策略与历史

- 发现策略列表（StrategyList）：展示已完成任务中 best_score > 0 的有效策略
  - 操作：保存到策略库 / 以此为种子继续进化（seed evolve）
- 算法日志（AlgorithmLog）：进化过程中的变异操作记录
- 历史探索表格（HistoryTable）：所有已完成或已停止的进化任务
  - 操作：查看任务详情 / 以此为种子继续进化
- 任务详情抽屉（TaskDetailDrawer）：单个进化任务的完整信息
  - 操作：种子进化 / 可视化验证（跳转到策略实验室回测模式）

#### 2.2.6 停止原因

| 停止原因             | 说明                |
|---------------------|---------------------|
| target_reached      | 达到目标分数         |
| stagnation          | 长期无改善（停滞）    |
| decline             | 分数持续下降         |
| max_generations     | 达到最大代数         |
| user_stop           | 用户手动停止         |
| error               | 运行异常             |
| invalid_dna         | 无效种子 DNA        |

### 2.3 策略库（`/strategies`）

策略库统一管理来自不同来源的策略。

**策略来源类型：** 实验室（lab） / 进化（evolution） / 导入（import）

**列表展示字段：**
- 名称（strategy name 或 strategy_id 前 8 位）
- DNA 摘要（signal_genes 前 2 个指标的参数概要）
- 收益率（best_score，颜色区分正/负/中性）
- 夏普值（best_score）
- 数据对（symbol / timeframe）
- 创建日期

**筛选与搜索：**
- 来源筛选下拉框：全部来源 / 实验室 / 进化 / 导入
- 搜索框：按策略名称、交易对、策略 ID 搜索

**操作功能：**
- 星标收藏：添加/取消星标（前端状态）
- 运行回测：跳转到实验室回测模式（预留）
- 编辑策略：修改策略信息（预留）
- 删除策略：弹出确认对话框（ConfirmDialog），确认后调用 API 删除
- 新建策略 / 导入策略入口按钮（预留）

### 2.5 数据管理（`/data`）

数据管理模块负责历史行情数据的导入、存储、质量检测和维护。

#### 2.5.1 数据集列表

- 展示字段：币种（symbol）、名称、周期（interval）、日期范围、K 线数量
- 数据质量指示条（QualityBar）：
  - 完整（complete）：绿色进度条 100%
  - 警告（warning）：黄色进度条 90%，缺口 < 5%
  - 错误（error）：红色进度条 60%，缺口 > 5%
  - 未知（unknown）：灰色进度条 0%
- 搜索功能：按币种或周期关键词搜索
- 操作按钮（悬浮显示）：预览 / 在实验室打开 / 删除

#### 2.5.2 CSV 文件上传（UploadDialog）

- 支持单文件和多文件批量上传（.csv 格式，支持多选）
- 自动从文件名识别 Symbol 和周期
  - 命名规则：`{SYMBOL}-{interval}-*.csv`（如 BTCUSDT-30m-2026-03.csv）
  - 正则匹配：`/^([A-Z]{3,10}USDT)-(\d+[mhdw])-/i`
  - 所有文件须 symbol 和 interval 一致才能自动识别
- 手动指定：Symbol 输入框、周期选择（15m / 30m / 1h / 4h / 1d / 3d / 1w）
- 导入模式：合并（merge）/ 替换（replace）/ 新建（new）
- 文件列表展示：文件名、文件大小、自动识别结果、删除按钮

#### 2.5.3 Binance 数据导入

## 3. 功能关联关系

### 3.1 数据流向

```
数据管理                 策略实验室              进化中心
(导入CSV)               (假设验证/回测)         (遗传算法)
    |                       |                       |
    +-- 提供行情数据 ------->|                       |
    |                       +-- 保存策略 ----------->|
    |                       |                       +-- 保存策略
    |                       |                       |
    |                       |<-- 可视化验证(带DNA) --+
    |                       |                       |
    +---------------------------------------------------------------> 模拟交易
    |                       |                       |               (策略运行)
    |                       v                       v
    |                   策略库 (统一存储、管理来自所有来源的策略)
    |                       |
    |                       +-- 来源: lab / evolution / import
    |                       +-- 操作: 运行回测 / 编辑 / 删除
    |
    +-- 设置 (指标参数通过 Zustand store 同步到策略实验室图表)
```

### 3.2 页面跳转关系

| 起点           | 终点           | 触发方式                        | 携带数据                                              |
|---------------|---------------|-------------------------------|------------------------------------------------------|
| 进化中心       | 策略实验室     | TaskDetailDrawer 中"可视化验证"按钮 | DNA, symbol, timeframe, dataStart, dataEnd            |
| 进化中心       | 进化中心       | "种子进化"按钮                   | 种子 DNA                                              |
| 进化中心       | 策略库         | "保存"策略按钮                   | name, DNA, symbol, timeframe, source=evolution        |
| 策略实验室     | 策略库         | "保存为策略"按钮                 | name, DNA, symbol, timeframe, source=lab              |
| 策略库         | 策略实验室     | "运行回测"按钮（预留）            | strategy_id 或 DNA                                    |
| 数据管理       | 策略实验室     | "在实验室打开"按钮（预留）         | dataset 信息                                          |
| 设置           | 数据管理       | 数据管理 Tab 提示                 | 无                                                    |

**跳转实现方式：**
- 进化中心 -> 策略实验室：使用 `navigate("/lab", { state: { dna, symbol, timeframe, ... } })`
- 策略实验室接收时通过 `location.state` 读取，并自动切换到回测模式
- 页面刷新后 route state 被清除（通过 `window.history.replaceState`），回退到假设验证模式

### 3.3 共享状态

- **数据源同步**：策略实验室和进化中心均通过 `GET /api/data/available-sources` 获取可用数据集列表，动态生成交易对和周期下拉选项
- **指标参数同步**：设置页面的指标配置通过 `useChartSettings`（Zustand store）全局共享，影响策略实验室的 K 线图表指标叠加
- **策略存储统一**：策略实验室和进化中心保存的策略统一写入策略库，通过 `source` 字段（lab / evolution / import）区分来源
- **进化进度实时同步**：进化中心通过 WebSocket（`/ws/evolution/{task_id}`）接收服务端推送

## 5. 用户操作流程

### 5.1 主流程：从数据导入到策略进化

```
步骤 1: 导入数据
  用户进入 [数据管理] 页面
  -> 点击 [上传 CSV 文件]
  -> 选择 CSV 文件（支持多选）
  -> 系统自动识别 Symbol 和周期（如 BTCUSDT-30m-2026-03.csv）
  -> 选择导入模式（合并/替换/新建）
  -> 点击 [上传]
  -> 后端解析 CSV、转换为 Parquet、记录元数据
  -> 数据集出现在列表中，质量检查完成

步骤 2: 手动验证假设（路径 A）
  用户进入 [策略实验室] 页面，假设验证模式
  -> 选择交易对和时间周期
  -> 设定日期范围（或使用快捷日期）
  -> 添加 WHEN 条件（如：RSI > 70）
  -> 添加 THEN 条件（如：close 下跌，窗口 12）
  -> 点击 [验证规律]
  -> 后端执行验证引擎 (POST /api/validate)
  -> 同时加载 K 线数据 (GET /api/data/ohlcv/{symbol}/{timeframe})
  -> 同时加载指标数据 (GET /api/data/chart-indicators/{symbol}/{timeframe})
  -> 查看验证结论、K 线标注、分布统计、触发记录
  -> 若符合率满意，点击 [保存为策略]
  -> 填写名称、描述、标签
  -> 前端生成 DNA (generateDnaFromValidation)
  -> 保存到策略库 (POST /api/strategies, source=lab)

步骤 3: 自动进化策略（路径 B）
  用户进入 [进化中心] 页面
  -> 选择探索模式（自动/种子）
  -> 配置参数（交易对/周期池/指标池/评分模板/种群/代数/杠杆/方向）
  -> 或使用快速预设（QuickPresets）
  -> 点击 [开始探索]
  -> 后端创建任务 (POST /api/evolution/tasks)
  -> 验证数据可用性（检查 Parquet 文件存在）
  -> 任务状态: pending -> running
  -> WebSocket 实时推送每代进度
  -> 观察分数趋势图和发现策略列表
  -> 可随时暂停/恢复/停止
  -> 达到目标分或最大代数后任务完成

步骤 4: 验证进化结果
  在 [进化中心] 的发现策略列表或任务详情抽屉中
  -> 点击 [可视化验证]
  -> navigate("/lab", { state: { dna, symbol, timeframe, dataStart, dataEnd } })
  -> 策略实验室切换到回测模式
  -> 自动执行回测 (POST /api/strategies/backtest)
  -> 查看回测指标、权益曲线、K 线信号标注
  -> 若满意，保存到策略库

步骤 5: 迭代优化
  在发现策略或历史探索中
  -> 点击 [种子进化]
  -> DNA 填充到种子探索配置
  -> 切换到种子模式，创建新进化任务
```

### 5.2 辅助流程

**策略管理流程：**
```
进入 [策略库] 页面
-> 通过搜索或来源筛选找到目标策略
-> 添加/取消星标

-> 配置成交量（显示位置）

```

## 7. 技术指标覆盖范围

系统支持在进化引擎和图表中使用以下技术指标，按五大类分组：

**趋势类（6 种）：** EMA（指数均线）、SMA（简单均线）、WMA（加权均线）、DEMA（双指数均线）、TEMA（三指数均线）、VWAP（成交量加权均价）

**动量类（9 种）：** RSI（相对强弱指数）、MACD（指数平滑异同移动平均线）、Stochastic（随机指标）、CCI（商品通道指数）、ROC（变动率）、Williams %R（威廉指标）、Aroon（阿隆指标）、CMO（钱德动量振荡）、TRIX（三重指数平滑）

**波动类（4 种）：** BB（布林带）、ATR（真实波幅）、Keltner（肯特纳通道）、Donchian（唐奇安通道）

**成交量类（9 种）：** OBV（能量潮）、CMF（蔡金资金流）、MFI（资金流量指数）、RVOL（相对成交量）、VROC（量变动率）、AD（累积分布）、CVD（累积成交量差）、VWMA（量加权均线）、VolumeProfile（成交量分布）

**趋势强度类（2 种）：** ADX（平均趋向指数）、PSAR（抛物线止损反转指标）

**图表指标计算支持（后端 `/api/data/chart-indicators`）：**
- EMA：支持自定义周期
- BB（布林带）：支持自定义周期和标准差
- RSI：支持自定义周期
- RVOL（相对成交量）：支持自定义周期
- VWMA（量加权均线）：支持自定义周期

## 8. DNA 策略编码结构

策略在系统中以 DNA（StrategyDNA）格式编码，这是贯穿策略实验室、进化中心和策略库的核心数据结构。

```
StrategyDNA
+-- signal_genes: SignalGene[]        信号基因数组
|   +-- indicator: string             指标名称（如 EMA、RSI）
|   +-- params: Dict                  指标参数（如 {period: 20}）
|   +-- role: SignalRole              角色（ENTRY_TRIGGER / EXIT_TRIGGER）
|   +-- condition: Dict               条件（如 {type: "price_above"}）
+-- logic_genes: LogicGenes           逻辑基因
|   +-- entry_logic: string           入场逻辑（AND / OR）
|   +-- exit_logic: string            出场逻辑（AND / OR）
+-- execution_genes: ExecutionGenes   执行基因
|   +-- timeframe: string             执行时间周期
|   +-- symbol: string                交易对
+-- risk_genes: RiskGenes             风控基因
    +-- stop_loss: float              止损比例
    +-- take_profit: float            止盈比例
    +-- position_size: float          仓位比例
    +-- leverage: int                 杠杆倍数
    +-- direction: string             方向（long / short）
```

DNA 支持多时间框架（MTF）模式，通过 layers 字段在不同周期上分配不同的指标组合和角色。
