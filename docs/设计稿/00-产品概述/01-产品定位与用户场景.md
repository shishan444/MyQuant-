## 1. 产品定位

### 1.1 产品名称

**MyQuant** -- BTC/ETH 量化交易策略进化工具

### 1.2 一句话定义

MyQuant 是一款面向个人加密货币交易者的桌面端量化工具，通过遗传算法从历史行情数据中自动进化交易策略，并提供可视化验证能力，帮助用户发现、理解并管理数据驱动的交易策略。

### 1.3 核心价值主张

传统量化工具存在两条路径：要么需要编程能力（如 Python + Backtrader），要么是黑盒信号推送（用户无法理解策略逻辑）。MyQuant 在两者之间开辟了第三条路径：

**用遗传算法从历史数据中进化策略，用可视化手段让用户理解并验证策略。**

具体而言：

- 用户无需编写代码，通过条件构建器（WHEN/THEN）即可表达交易假设
- 系统通过遗传算法自动探索策略空间，发现人类难以直觉发现的组合
- 每个策略以 DNA 结构（信号基因 + 逻辑基因 + 执行基因 + 风控基因）呈现，可解释、可追溯
- K 线图 + 信号标注 + 资金曲线提供完整的视觉验证闭环

### 1.4 目标市场与用户

| 维度 | 说明 |
|------|------|
| 目标用户 | 有 1-3 年 BTC/ETH 交易经验的个人交易者 |
| 用户画像 | 看过技术分析书籍、了解 RSI/EMA/MACD 等指标，但没有编程能力；希望用数据验证自己的交易直觉，或发现新的策略机会 |
| 核心痛点 | 交易决策依赖主观判断，无法量化验证；知道某些指标有用但不知道如何组合；市面上的量化工具门槛太高 |
| 使用场景 | 个人电脑本地部署，浏览器访问；主要用于策略研究和回测验证 |
| 支持品种 | BTC/USDT、ETH/USDT |
| 支持周期 | 15m / 30m / 1h / 4h / 1d / 3d |

### 1.5 与竞品的差异

| 维度 | MyQuant | TradingView | FMZ 量化 |
|------|---------|-------------|---------|
| 策略生成方式 | 遗传算法自动进化 | 用户手动编写 Pine Script | 用户手动编写 Python/JS |
| 编程门槛 | 零代码 | 需学习 Pine Script | 需掌握编程语言 |
| 策略可解释性 | DNA 结构 + 信号基因可视化 | 代码即策略 | 代码即策略 |
| 验证方式 | K 线标注 + 资金曲线 + 指标面板 | 策略测试器 | 回测引擎 |
| 策略发现 | 自动探索策略空间 | 纯手动 | 纯手动 |
| 部署方式 | 本地部署 | 云端 SaaS | 云端 + 本地 |
| 实盘能力 | 当前仅模拟 | 支持 | 支持 |

## 2. 核心用户场景

### 2.1 场景一：假设驱动验证（Hypothesis-Driven Validation）

**场景描述：**

用户在日常交易中观察到一个价格规律，希望用历史数据验证这个规律是否成立。

**典型流程：**

1. 用户打开 MyQuant，默认进入 Lab 页面
2. 在"假设验证"模式下，选择交易对（BTCUSDT）和周期（4h）
3. 设置 WHEN 条件："RSI 低于 30"（subject=RSI, action=below, target=30）
4. 设置 THEN 条件："close 上涨 3%"（subject=close, action=rise, target=3, window=12）
5. 选择时间范围，点击"验证"
6. 系统在历史数据中扫描所有满足 WHEN 条件的触发点
7. 结果展示：
   - 触发点表格：列出每次触发的具体时间、价格、是否满足 THEN
   - 分布图：横轴为涨跌幅区间，纵轴为触发次数
   - K 线图：标注所有触发点，用户可逐个查看上下文
   - 验证结论：匹配率、百分位数、集中度等统计指标
8. 用户根据结论决定是否将该假设转化为可执行的交易策略

**场景价值：**

- 将模糊的交易直觉转化为可量化的假设
- 用历史数据快速验证，避免主观偏差
- 从"我觉得"升级为"数据显示"

**入口页面：** Lab (/lab) -- 假设验证模式

---

### 2.2 场景二：进化驱动发现（Evolution-Driven Discovery）

**场景描述：**

用户不想手动设计策略，而是希望系统自动探索并发现有效的交易策略组合。

**典型流程：**

1. 用户进入进化中心 (/evolution)
2. 选择"自动探索"模式（Auto 模式）
3. 配置探索参数：
   - 交易对：BTCUSDT
   - 时间周期池：4h + 1d（多周期联动）
   - 指标池：EMA, RSI, MACD, Bollinger Bands（选择参与进化的指标）
   - 进化参数：种群规模 15, 最大代数 200, 目标评分 80
   - 杠杆与方向：1x, long
4. 点击"启动进化"
5. 系统创建进化任务，通过 WebSocket 实时推送进度
6. 进化过程展示：
   - 进度条：当前代数 / 最大代数
   - 评分趋势图：每代最优评分与平均评分的折线图
   - 发现的策略列表：实时更新最优个体的 DNA 结构和评分
   - 算法日志：记录关键变异操作和评分变化
7. 进化完成后，用户查看最终 Champion 策略的详细信息
8. 用户选择将 Champion 策略发送到 Lab 进行视觉验证（进入场景三）

**变体 -- 种子模式（Seed 模式）：**

用户已经有一个初步策略（可能来自场景一的验证结果），希望以此为基础进行优化。用户切换到"种子进化"模式，提供初始 DNA，系统在此基础上进行遗传操作，探索更好的参数组合和结构变体。

**场景价值：**

- 自动探索人类难以穷举的指标组合空间
- 遗传算法提供结构化的搜索策略，避免随机试错
- 种子模式支持"人机协作"：人类提供直觉，算法提供优化

**入口页面：** Evolution (/evolution)

---

### 2.3 场景三：跨模块流转（Cross-Module Flow）

**场景描述：**

用户在进化中心发现了一个有潜力的策略，需要通过完整的视觉验证流程确认其有效性，然后保存到策略库。

**典型流程：**

1. 进化中心发现了一个评分 82 的策略
2. 用户点击"查看回测"按钮
3. 系统将该策略的 DNA 通过 URL 参数传递到 Lab 的回测模式
4. Lab 页面自动切换到"回测模式"，加载 DNA 并执行回测
5. 回测结果展示：
   - K 线图：标注买入/卖出信号点，支持缩放和平移
   - 资金曲线：独立图表展示资金随时间的变化
   - 指标面板：总收益率、夏普比率、最大回撤、胜率、交易次数、综合评分
   - DNA 详情：策略的完整基因结构展示
6. 用户在 K 线图上逐一检查信号点的合理性：
   - 买入点是否在合理位置
   - 是否存在频繁交易的噪音
   - 大幅回撤发生在什么行情阶段
7. 确认满意后，点击"保存策略"
8. 系统弹出保存对话框，用户填写策略名称、标签和备注
9. 策略保存到策略库，可在策略库页面查看和管理

**场景价值：**

- 从发现到验证到保存的完整闭环
- 视觉验证弥补了纯数值评分的不足
- 策略库提供统一的策略管理，方便后续对比和使用

**涉及页面：** Evolution -> Lab -> Strategies

### 3.2 当前限制

| 限制项 | 说明 |
|--------|------|
| 无实盘交易 | 当前仅支持回测和模拟，不支持连接真实交易所下单 |
| 模拟交易页面为占位 | Trading 页面使用 Mock 数据展示，未接入真实数据源 |
| 品种限制 | 仅支持 BTC/ETH，不支持其他币种 |
| 数据源依赖 | 需要用户自行从 Binance 导出 CSV 数据，无自动数据获取 |
| 单机部署 | 无用户系统和多租户支持，仅限单用户本地使用 |
| 无 Walk-Forward 前端 | 后端支持 Walk-Forward 验证，但前端未提供配置入口 |
| 策略对比功能未完整 | 后端支持多策略对比 API，前端对比界面待完善 |
| 无自动定时任务 | 进化任务需手动启动，不支持定时/周期性运行 |

## 4. 产品设计原则

### 4.1 零代码优先

所有核心功能通过图形界面操作完成，用户不需要编写任何代码。条件构建器、配置表单、拖拽操作是主要的交互方式。

### 4.2 可解释性优先

策略不是黑盒。每个策略都以 DNA 结构展示，用户可以看到使用了哪些指标、什么参数、什么逻辑关系、什么风控参数。进化过程中也记录了每一步变异操作。

### 4.3 视觉验证优先

数值指标（收益率、夏普比率）很重要，但不足以让用户建立信心。K 线图上的信号标注、资金曲线的趋势、分布图的形态，这些视觉信息帮助用户真正理解策略的行为。

### 4.4 渐进式复杂度

用户可以从最简单的假设验证开始（场景一），不需要理解遗传算法。随着信任建立，逐步使用进化发现（场景二）和跨模块流转（场景三）等高级功能。

## 6. 术语表

| 术语 | 定义 |
|------|------|
| DNA | StrategyDNA，策略的基因编码结构，包含信号基因、逻辑基因、执行基因、风控基因 |
| SignalGene | 信号基因，单个指标条件，如 RSI(14) < 30 |
| LogicGenes | 逻辑基因，信号之间的组合逻辑（AND/OR） |
| ExecutionGenes | 执行基因，交易对和时间周期 |
| RiskGenes | 风控基因，止损、止盈、仓位、杠杆、方向 |
| Champion | 进化任务中表现最优的策略个体 |
| WHEN 条件 | 假设验证中的触发条件，描述"什么情况下" |
| THEN 条件 | 假设验证中的结果条件，描述"之后会发生什么" |
| 触发点 | 历史数据中满足 WHEN 条件的时间点 |
| 匹配率 | 在所有触发点中，THEN 条件也满足的比例 |
| Walk-Forward | 时序交叉验证方法，将数据分为训练窗口和验证窗口滑动验证 |
| 种群 | 一代进化中所有策略个体的集合 |
| 代数 | 遗传算法的迭代轮次 |
| 变异 | 对 DNA 结构的随机修改操作（参数变异、指标替换等） |
| 交叉 | 两个父代 DNA 组合产生子代的操作 |
| 精英保留 | 每代保留一定比例的最优个体直接进入下一代 |
| 多周期（MTF） | Multi-Timeframe，策略同时在多个时间周期的数据上运行 |
