# 03 - 进化中心 (Evolution)

## 1. 概述

进化中心是 MyQuant 的策略自动发现引擎前端界面。用户配置参数后, 系统通过遗传算法在策略空间中搜索最优解。
页面提供探索配置、实时进度监控、策略发现、历史查看等功能。

文件位置: `src/pages/Evolution.tsx`

## 2. 页面区域划分

```
+-----------------------------------------------------+
| Area 1: 探索配置 (GlassCard)                          |
|  - SegmentedControl (自动/种子模式)                   |
|  - AutoConfigForm / SeedConfigForm                   |
|  - 折叠/展开切换                                      |
+-----------------------------------------------------+
| Area 2: 探索进度 (GlassCard)                          |
|  - ProgressPanel (进度/分数/控制)                     |
|  - ScoreTrendChart (代际趋势图)                       |
|  - QuickPresets (无任务时展示)                        |
+-----------------------------------------------------+
| Area 3: 发现策略 (GlassCard, 条件渲染)                |
|  - StrategyList                                      |
+-----------------------------------------------------+
| Area 4: 算法日志 (GlassCard, 条件渲染)                |
|  - AlgorithmLog                                      |
+-----------------------------------------------------+
| Area 5: 历史探索 (GlassCard, 条件渲染)                |
|  - HistoryTable                                      |
+-----------------------------------------------------+
```

## 3. 探索配置 (Area 1)

### 3.1 模式选择

使用 `SegmentedControl` 组件切换两种探索模式:

- **自动模式 (auto)**: 系统从零开始, 随机生成初始种群并进化
- **种子模式 (seed)**: 用户提供初始 DNA, 在其基础上进化

### 3.2 AutoConfigForm

文件位置: `src/components/evolution/AutoConfigForm.tsx`

配置项:

| 字段              | 类型                     | 默认值     | 说明                              |
|------------------|--------------------------|-----------|-----------------------------------|
| symbol           | string                   | -         | 交易对选择, 从可用数据源动态获取    |
| timeframePool    | string[]                 | -         | 时间周期组合, 有序列表              |
| indicatorPool    | string[]                 | -         | 指标池, 勾选                       |
| scoreTemplate    | string                   | profit_first | 评分模板                        |
| populationSize   | number                   | 15        | 种群大小                           |
| maxGenerations   | number                   | 30        | 最大代数                           |
| targetScore      | number                   | 80        | 目标分数                           |
| leverage         | number                   | 1         | 杠杆倍数                           |
| direction        | long/short/mixed         | long      | 交易方向                           |
| dataStart        | string                   | -         | 数据起始日期                       |
| dataEnd          | string                   | -         | 数据结束日期                       |

时间周期组合设计:

周期池是一个有序列表, 按角色标注:

| 位置 | 角色   | 说明               |
|-----|--------|--------------------|
| 0   | 趋势   | 最长周期, 判断大方向 |
| 1   | 判断   | 中等周期, 信号确认   |
| 2   | 确认   | 较短周期, 辅助确认   |
| 3   | 执行   | 最短周期, 入场执行   |

角色标签定义: `TF_LAYER_ROLES = ["趋势", "判断", "确认", "执行"]`

指标池分组:

```
趋势: EMA, SMA, WMA, DEMA, TEMA, VWAP
动量: RSI, MACD, Stochastic, CCI, ROC, Williams %R, Aroon, CMO, TRIX
波动: BB, ATR, Keltner, Donchian
成交量: OBV, CMF, MFI, RVOL, VROC, AD, CVD, VWMA, VolumeProfile
趋势强度: ADX, PSAR
```

评分模板选项:

| 值             | 标签     | 权重分配                               |
|---------------|---------|----------------------------------------|
| profit_first  | 收益优先 | 年化35% / 夏普25% / 回撤25% / 胜率15%  |
| steady        | 稳健优先 | 年化20% / 夏普35% / 回撤35% / 卡玛10%  |
| risk_first    | 风控优先 | 年化10% / 夏普30% / 回撤40% / 卡玛20%  |

### 3.3 SeedConfigForm

文件位置: `src/components/evolution/SeedConfigForm.tsx`

与 AutoConfigForm 类似, 但增加:

- **initialDna**: 初始 DNA 输入, 可从其他策略或进化结果填入
- 无 timeframePool (使用 DNA 中的 execution_genes.timeframe)
- 无 indicatorPool (从 DNA 推断)

种子 DNA 来源:

1. 从发现策略列表点击 "种子进化"
2. 从历史探索记录点击 "种子进化"
3. 从 TaskDetailDrawer 点击 "种子进化"

### 3.4 配置折叠行为

配置区域在任务运行期间自动折叠, 任务结束或无任务时自动展开:

- 折叠状态: 显示一行摘要 (symbol + timeframe + mode + score_template + 进度)
- 展开状态: 显示完整配置表单
- 手动折叠/展开: 通过 "展开配置" 链接

自动折叠逻辑:

```
有活跃任务 + 配置展开 -> 自动折叠
无活跃任务 + 配置折叠 -> 自动展开
```

## 4. 探索进度 (Area 2)

### 4.1 进度面板

文件位置: `src/components/evolution/ProgressPanel.tsx`

展示内容:

1. **状态栏**: 状态指示灯 (运行中蓝色闪烁/暂停琥珀色/完成绿色) + 状态文字 + 已探索组合数 + 代数进度
2. **进度条**: 百分比进度, 达到目标分数时变绿
3. **分数对比**: 两个 StatCard 并排展示当前最优和目标分数
4. **诊断标签**: 数据行数 (K线条数) / 杠杆倍数 / 交易方向
5. **控制按钮**: 暂停/恢复 + 停止

控制按钮逻辑:

| 状态     | 显示按钮             |
|---------|----------------------|
| running | 暂停 + 停止          |
| paused  | 恢复 + 停止          |
| 其他    | 不显示               |

停止按钮使用 ConfirmDialog 确认, 描述中包含任务 ID 前 8 位, 确认不会丢失已发现策略。

### 4.2 ScoreTrendChart (代际趋势图)

文件位置: `src/components/evolution/ScoreTrendChart.tsx`

使用 Recharts 的 ComposedChart (折线 + 散点):

- **最优分数线**: 琥珀色实线 (`#eab308`), 线宽 2
- **平均分数线**: 紫色虚线 (`rgba(168,85,247,0.4)`), 线宽 1
- **目标线**: 灰色虚线水平参考线, 标注目标值
- **最优更新点**: 绿色菱形散点, 标记分数提升的代际

数据来源: `useEvolutionHistory(activeTaskId)` 返回的 `EvolutionHistoryRecord[]`

图表高度: 200px

底部统计:

- 最优刷新次数
- 最长停滞代数 (>10 代显示橙色警告)
- Fallback 百分比 (非零时显示红色警告)

### 4.3 QuickPresets (快捷预设)

无活跃任务时展示, 提供一键创建典型探索任务的快捷入口。

## 8. 实时通信 (WebSocket)

### 8.1 useEvolutionWebSocket

文件位置: `src/hooks/useEvolution.ts`

WebSocket 连接管理:

- 连接地址: `ws(s)://{host}/ws/evolution/{taskId}`
- 协议选择: 根据当前页面协议自动选择 ws/wss
- Host 来源: 优先 `VITE_WS_URL` 环境变量, 回退 `window.location.host`

消息处理:

| 消息类型               | 处理方式                              |
|----------------------|---------------------------------------|
| generation_complete  | 更新 task query cache (代数/分数/DNA) |
| evolution_complete   | 更新状态为 completed                   |
| 其他                  | 合并更新到 task cache                  |

缓存更新策略:

1. `generation_complete` 时直接通过 `setQueryData` 更新 task 缓存, 避免重新请求
2. 2 秒延迟批量触发 `invalidateQueries`, 刷新 history 和 strategies 查询

断线重连:

- 连接关闭后 3 秒自动重连
- 组件卸载时清理连接和定时器

### 8.2 数据流

```
WebSocket 推送 generation_complete
    |
    v
更新 React Query Cache (task)
    |
    v (2s debounce)
invalidateQueries (history, strategies)
    |
    v
UI 自动刷新 (ProgressPanel, ScoreTrendChart, StrategyList)
```

## 9. 任务状态管理

### 9.1 任务生命周期

```
创建任务 -> pending -> running -> completed
                  |-> paused -> running (恢复)
                  |-> stopped
```

状态判断:

```typescript
function isActiveStatus(status: EvolutionTaskStatus): boolean {
  return status === "running" || status === "pending";
}
```

### 9.2 并发控制

- 系统同一时间只允许一个活跃任务
- 创建新任务前检查: `if (activeTask) toast.error("已有正在进行的探索任务")`
- 活跃任务存在时, 配置表单 disabled

### 9.3 停止原因映射

| 值               | 标签              |
|-----------------|-------------------|
| target_reached  | 达到目标分数       |
| stagnation      | 长期无改善(停滞)   |
| decline         | 分数持续下降       |
| max_generations | 达到最大代数       |
| user_stop       | 用户手动停止       |
| error           | 运行异常          |
| invalid_dna     | 无效种子DNA       |

## 11. 设计约束

1. 同一时间仅允许一个活跃探索任务, 新建前必须停止或等待当前任务完成
2. WebSocket 断线后自动重连, 不丢失已接收的代际数据
3. 配置区域在任务运行时自动折叠, 节省屏幕空间, 但可手动展开
4. 发现策略列表仅展示 best_score > 0 的已完成/已停止任务
5. 停止操作需二次确认, 确保用户了解已发现策略不会被删除
6. 可视化验证跳转到 Lab 页面, 通过路由 state 传递 DNA 和参数, 不经过全局状态
