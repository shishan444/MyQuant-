# 01 - 页面与路由设计

## 1. 概述

MyQuant 前端采用单页应用(SPA)架构, 基于 React 18 + react-router v7 构建。应用共包含 6 个功能页面,
统一由 AppLayout 布局组件包裹, 侧边栏提供全局导航。

## 2. 路由配置

### 2.1 路由定义

路由在 `App.tsx` 中通过 `createBrowserRouter` 集中定义:

| 路径         | 组件             | 说明                 |
|-------------|-----------------|----------------------|
| `/`         | `Navigate`      | 重定向至 `/lab`       |
| `/lab`      | `Lab`           | 策略实验室(默认着陆页) |
| `/evolution`| `Evolution`     | 进化中心              |
| `/strategies`| `Strategies`    | 策略库               |
| `/trading`  | `Trading`       | 模拟交易              |
| `/data`     | `DataManagement`| 数据管理              |
| `/settings` | `Settings`      | 设置                  |

### 2.2 路由结构

```
createBrowserRouter([
  {
    element: <AppLayout />,
    children: [
      { index: true, element: <Navigate to="/lab" replace /> },
      { path: "/lab", element: <Lab /> },
      { path: "/evolution", element: <Evolution /> },
      { path: "/strategies", element: <Strategies /> },
      { path: "/trading", element: <Trading /> },
      { path: "/data", element: <DataManagement /> },
      { path: "/settings", element: <Settings /> },
    ],
  },
])
```

根路径 `/` 使用 `<Navigate to="/lab" replace />` 做服务端跳转, 确保用户直接访问域名时落地到策略实验室。

### 2.3 路由器初始化

`QueryClient` 在路由器之外创建, 实例化参数:

- `staleTime`: 30000ms (30秒)
- `retry`: 1 (失败重试一次)

`QueryClientProvider` 包裹 `RouterProvider`, 确保所有路由组件均能访问 React Query 上下文。

```typescript
const queryClient = new QueryClient({
  defaultOptions: {
    queries: { staleTime: 30_000, retry: 1 },
  },
});
```

## 3. 全局布局

### 3.1 AppLayout

文件位置: `src/components/layout/AppLayout.tsx`

布局结构:

```
+-----------------------------------------------------------+
| Sidebar | Header                                           |
|         +--------------------------------------------------+
|         | main (scrollable, padding: 24px)                 |
|         |                                                  |
|         | <Outlet /> (当前路由组件渲染于此)                    |

```

组件结构:

```

  <Sidebar />
  <div className="flex flex-1 flex-col overflow-hidden">

    <main className="flex-1 overflow-auto p-6">
```

关键特征:

- 整体高度锁定视口 (`h-screen`), 内容区可滚动 (`overflow-auto`)
- Sidebar 固定左侧, 宽度可折叠 (240px / 64px)
- Header 固定顶部, 高度 48px (`h-12`)
- Toaster 使用 sonner 库, 主题为 dark, 位置右上角, 宽度 360px

```typescript
const NAV_ITEMS = [
```

## 4. 页面间状态传递

### 4.1 Evolution -> Lab (DNA 传递)

进化中心到策略实验室的数据传递使用 react-router 的 `navigate` state 机制:

```typescript
// Evolution 页面发起导航
navigate("/lab", {
  state: {
    dna,
    symbol: task.symbol,
    timeframe: task.timeframe,
    dataStart: task.data_start || task.data_time_start?.slice(0, 10) || undefined,
    dataEnd: task.data_end || task.data_time_end?.slice(0, 10) || undefined,
  },
});
```

Lab 页面接收:

```typescript
const location = useLocation();
const routeState = location.state as {
  dna?: DNA;
  symbol?: string;
  timeframe?: string;
  dataStart?: string;
  dataEnd?: string;
} | null;
```

状态消费逻辑:

1. 检测 `routeState?.dna` 是否存在
2. 存在则切换到回测模式 (`labMode = "backtest"`)
3. 不存在则保持假设验证模式 (`labMode = "hypothesis"`)
4. 页面加载后立即清除路由状态 (`window.history.replaceState({}, "")`), 确保刷新后回到假设验证模式

### 4.2 状态传递字段说明

| 字段         | 类型     | 来源              | 用途                          |
|-------------|---------|-------------------|-------------------------------|
| `dna`       | `DNA`   | EvolutionTask 的 champion_dna | 策略基因, 用于回测             |
| `symbol`    | `string`| EvolutionTask.symbol | 交易对, 如 "BTCUSDT"         |
| `timeframe` | `string`| EvolutionTask.timeframe | 时间周期, 如 "4h"          |
| `dataStart` | `string`| EvolutionTask.data_start | 数据起始日期, YYYY-MM-DD    |
| `dataEnd`   | `string`| EvolutionTask.data_end | 数据结束日期, YYYY-MM-DD      |

### 4.3 其他页面间通信

- 全局状态通过 Zustand stores 共享 (app, lab, chart-settings)
- 服务端数据通过 React Query 缓存共享, 使用统一的 query key 命名空间
- 通知消息通过 sonner Toaster 全局展示

## 5. 页面过渡动画

所有页面组件被 `PageTransition` 包裹:

文件位置: `src/components/PageTransition.tsx`

- 使用 framer-motion 实现
- 进入动画: opacity 0 -> 1, y 8px -> 0
- 持续时间: 200ms
- 不设退出动画, 避免路由切换闪烁


## 8. 设计约束

1. 所有页面共享同一 AppLayout, 无嵌套布局
2. 路由层不包含鉴权逻辑 (当前版本无用户系统)
3. 不使用路由级别的数据预加载 (loader), 数据获取在各页面组件内通过 React Query 完成
4. 路由 state 仅用于跨页面一次性数据传递, 不作为持久状态存储
5. 目标视口: 1440 x 900 (桌面端优先)
