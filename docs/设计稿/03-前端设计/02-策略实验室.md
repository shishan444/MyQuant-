# 02 - 策略实验室 (Lab)

## 1. 概述

策略实验室是 MyQuant 的核心交互页面, 承载两大功能: 假设验证与策略回测。页面通过顶部模式切换器在两种模式间切换。
默认进入假设验证模式; 当从进化中心携带 DNA 跳转时, 自动进入回测模式。

文件位置: `src/pages/Lab.tsx`

## 2. 模式切换

### 2.1 切换器 UI

页面顶部提供两个按钮, 使用类似 SegmentedControl 的样式:

- "假设验证" (FlaskConical 图标)
- "策略回测" (BarChart3 图标)

激活态: 琥珀色背景 (`bg-amber-400/15`) + 琥珀色文字
未激活态: 灰色文字 (`text-slate-500`), 悬浮变亮

### 2.2 模式决定逻辑

```typescript
type LabMode = "hypothesis" | "backtest";

// 初始模式由路由状态决定
const [labMode, setLabMode] = useState<LabMode>(
  routeState?.dna ? "backtest" : "hypothesis"
);
```

模式切换规则:

| 条件                      | 默认模式     |
|--------------------------|-------------|
| 直接访问 `/lab`           | 假设验证     |
| 从进化中心携带 DNA 跳转    | 策略回测     |
| 用户手动点击切换器         | 用户选择     |

路由状态清理: 页面首次加载后执行 `window.history.replaceState({}, "")`,
确保刷新后回到假设验证模式。

## 3. 假设验证模式

### 3.1 页面区域划分

```
+-----------------------------------------------------+
| Region 1: 规律构建器 (GlassCard)                      |
|  - 数据源选择 (交易对/周期/日期)                       |
|  - WHEN 条件组                                        |
|  - THEN 条件组                                        |
|  - 操作按钮 (清空/验证/保存)                           |
+-----------------------------------------------------+
| Region 2: 验证结论 (ValidationConclusion)             |
+-----------------------------------------------------+
| Region 3: K 线图 (KlineChart + 触发标记)              |
+-----------------------------------------------------+
| Region 4: 分布图 + 参考面板                           |
|  - DistributionChart (左半)                           |
|  - ReferencePanel (右半)                              |
+-----------------------------------------------------+
| Region 5: 触发表 (TriggerTable)                       |
+-----------------------------------------------------+
```

### 3.2 规律构建器

#### 数据源选择

控件布局为一行, 包含:

- **交易对选择器** (Select): 从 `useAvailableSources` 动态获取可用交易对,
  回退到静态 `SYMBOL_OPTIONS` (BTC/ETH/BNB/SOL)
- **时间周期选择器** (Select): 根据选中交易对过滤可用周期,
  回退到静态 `TIMEFRAME_SELECT_OPTIONS` (15m/1h/4h/1d/1w)
- **日期范围**: 两个 `<input type="date">`, 默认近一年
- **快捷日期**: 近3月 / 近6月 / 近1年 / 今年 / 全部

日期处理:

- `toLocalDateStr()`: 将 Date 对象格式化为本地时区 YYYY-MM-DD, 避免时区偏移
- `clampDate()`: 将非法日期 (如 04-31) 修正为合法日期 (04-30)
- 自动根据数据源的实际时间范围设置默认日期区间

#### WHEN 条件组 (ConditionPillGroup)

- 标签: "WHEN"
- 描述: '找出"什么情况下"'
- 每个条件包含: subject (指标), action (动作), target (目标值), logic (AND/OR), timeframe (可选跨周期)
- 支持添加/删除/编辑条件

#### THEN 条件组 (ConditionPillGroup)

- 标签: "THEN"
- 描述: '验证"触发后会发生什么"'
- 增加 `isThen` 属性, 支持 window 参数 (回溯窗口)
- 其余结构与 WHEN 一致

#### 操作按钮

- "清空全部" (RotateCcw 图标): 清空所有条件和结果
- "验证规律" (Play 图标): 提交验证请求, 需要 WHEN 和 THEN 各至少一个条件
- "保存为策略" (Save 图标): 仅在有结果且 total_count > 0 时显示

#### 构建器折叠

构建器支持折叠/展开, 折叠时显示条件摘要:

```
BTCUSDT 4h / WHEN: close touch bb_upper / THEN: close drop 0
```

### 3.3 示例预设

页面空状态时展示 3 个示例预设按钮:

| 序号 | 名称            | WHEN 条件                        | THEN 条件                       |
|-----|-----------------|----------------------------------|---------------------------------|
| 1   | 上轨触碰后下跌   | close touch bb_upper             | close drop 0 (window=8)         |
| 2   | 放量突破后上涨   | close breakout bb_upper + volume spike 2 | close rise 0 (window=8) |
| 3   | RSI超买后回调   | rsi gt 70                        | close drop 0 (window=12)        |

点击预设按钮自动填充 WHEN/THEN 条件。

### 3.4 验证流程

```
用户点击 "验证规律"
    |
    v
前端校验 (日期有效性, 条件完整性)
    |
    v
调用 validateHypothesis API (POST /api/validate)
    |
    +--> 并行请求:
    |    1. getOhlcvBySymbol (OHLCV 数据)
    |    2. getChartIndicators (指标数据, EMA/BOLL/RSI)
    |
    v
渲染结果:
    - ValidationConclusion (结论面板)
    - KlineChart (K 线 + 触发标记)
    - DistributionChart (幅度区间分布)
    - ReferencePanel (统计参考)
    - TriggerTable (触发记录表)
```

### 3.5 验证结果展示

#### ValidationConclusion (验证结论面板)

展示核心指标:

- 符合率 (`match_rate`)
- 总触发次数 (`total_count`)
- 符合次数 (`match_count`) / 不符合次数 (`mismatch_count`)

#### K 线图与触发标记

K 线图高度 650px, 叠加触发标记:

- 符合的触发点: 绿色圆圈 (`#00C853`), 显示在 K 线高点上方 1.2% 处
- 不符合的触发点: 红色圆圈 (`#EF4444`)
- 图表工具栏: 缩放/重置/全屏
- 图例: 显示 OHLCV、EMA、BOLL、触发点分组

#### DistributionChart (幅度区间分布)

- 基于 Recharts 的堆叠柱状图
- X 轴: 幅度区间 (如 -5%~-3%)
- Y 轴: 触发次数
- 绿色: 符合 (`rgba(0,200,83,0.5)`)
- 红色: 不符合 (`rgba(239,68,68,0.5)`)

#### ReferencePanel (参考面板)

展示辅助统计:

- `percentiles`: 百分位数据
- `concentration`: 集中度分布
- `signal_frequency`: 信号频率
- `extremes`: 极值记录

#### TriggerTable (触发表)

展示每条触发记录:

- 触发时间、价格、涨跌幅
- 是否符合 (matched)
- 指标值快照

操作:

- "定位"按钮: 调用 `chartRef.current.scrollToTime()` 滚动到对应位置
- "详情"按钮: 打开 TriggerDetailDrawer

#### TriggerDetailDrawer

侧边抽屉, 展示单条触发记录的详细信息:

- 触发时间、价格
- 涨跌幅
- 各指标在触发时刻的具体值

### 3.6 保存策略

点击 "保存为策略" 打开 SaveStrategyDialog:

1. 调用 `generateDnaFromValidation()` 将 WHEN/THEN 条件转换为 DNA
2. 用户填写: 策略名称、描述、标签
3. 调用 `createStrategy` API (POST /api/strategies)

## 4. 策略回测模式

### 4.1 入口条件

- 从进化中心携带 DNA 跳转: 直接进入并自动运行回测
- 手动切换: 显示空状态提示 "请从进化中心选择一个策略进行可视化验证"

### 4.2 BacktestModePanel 组件

文件位置: `src/components/lab/BacktestModePanel.tsx`

Props:

```typescript
interface BacktestModePanelProps {
  initialDna: DNA;
  initialSymbol: string;
  initialTimeframe: string;
  initialDataStart?: string;
  initialDataEnd?: string;
}
```

#### 自动回测流程

```
组件挂载
    |
    v
并行请求:
    1. runBacktest API (POST /api/strategies/backtest)
    2. getOhlcvBySymbol (K 线数据)
    |
    v
结果渲染:
    - StrategyDetail (DNA 概览)
    - KlineChart + 交易信号 (买入/卖出标记)
    - EquityCurveChart (资金曲线)
    - BacktestMetricsPanel (回测指标)
```

#### 回测结果展示

**策略基因**: 使用 `StrategyDetail` 组件展示 DNA 结构概要。

**K 线 + 交易信号**:

- 高度 420px
- 买入信号: 绿色向上箭头 (belowBar)
- 卖出信号: 红色向下箭头 (aboveBar)
- 底部显示买卖计数和数据范围

**资金曲线**:

- 使用 lightweight-charts 的 LineSeries
- 琥珀色线条 (`#f59e0b`), 线宽 1.5
- 显示初始资金和最终资金
- 爆仓时显示红色警告文字

**回测指标** (BacktestMetricsPanel):

核心指标项:

| 指标              | 字段             | 说明             |
|------------------|------------------|-----------------|
| 总收益率          | `total_return`   | 百分比           |
| 夏普比率          | `sharpe_ratio`   | 风险调整收益     |
| 最大回撤          | `max_drawdown`   | 百分比           |
| 胜率              | `win_rate`       | 百分比           |
| 总交易次数        | `total_trades`   | 整数             |
| 总评分            | `total_score`    | 综合得分         |
| 资金费成本        | `total_funding_cost` | 累计资金费   |
| 是否爆仓          | `liquidated`     | 布尔值           |

#### 操作按钮

- "重新运行": 重新执行回测
- "保存策略": 将 DNA 保存到策略库

## 5. DNA 生成

### 5.1 generateDnaFromValidation

文件位置: `src/lib/dna-generator.ts`

将假设验证的 WHEN/THEN 条件映射为 DNA 结构:

映射规则:

| 条件来源 | DNA 角色          |
|---------|-------------------|
| WHEN    | `entry_trigger`   |
| THEN    | `exit_trigger`    |

条件类型映射:

| 条件动作      | DNA 条件类型        |
|--------------|---------------------|
| gt           | gt                  |
| lt           | lt                  |
| touch        | price_above         |
| cross_above  | cross_above         |
| cross_below  | cross_below         |
| spike        | gt                  |
| breakout     | cross_above         |
| rise         | gt                  |
| drop         | lt                  |

默认参数:

- RSI: period=14
- EMA: period=20
- MACD: fast=12, slow=26, signal=9
- entry_logic: AND
- exit_logic: AND
- stop_loss: 0.05 (5%)
- take_profit: 0.1 (10%)
- position_size: 0.3
- leverage: 1
- direction: long

## 6. 关键交互流程

### 6.1 假设验证完整流程

1. 用户选择交易对和时间周期
2. 设置日期范围 (或使用快捷按钮)
3. 添加 WHEN 条件 (至少 1 个)
4. 添加 THEN 条件 (至少 1 个)
5. 点击 "验证规律"
6. 等待验证完成 (显示 loading skeleton)
7. 查看结论、K 线图、分布图、触发表
8. 可选: 点击触发表行定位到 K 线图
9. 可选: 保存为策略

### 6.2 从进化中心进入回测

1. 在进化中心点击 "可视化验证"
2. 路由跳转到 `/lab` 并携带 state
3. Lab 自动切换到回测模式
4. BacktestModePanel 自动运行回测
5. 展示 K 线 + 信号、资金曲线、指标面板
6. 可选: 保存策略到策略库

## 8. 设计约束

1. 假设验证模式与回测模式互斥, 同一时间只显示一种模式的 UI
2. 回测模式必须有 DNA 输入, 无 DNA 时显示空状态引导
3. 路由 state 为一次性消费, 刷新后回到假设验证模式
4. 日期范围自动根据数据源可用范围调整
5. 图表指标 (EMA/BOLL/RSI) 由后端计算, 前端仅做渲染
6. 触发标记使用透明 LineSeries 偏移实现 (offset = high * 1.012), 确保不遮挡 K 线
