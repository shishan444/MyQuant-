## 1. 后端技术栈

### 1.1 运行时与框架

| 技术             | 版本要求          | 用途                                       |
|----------------|---------------|------------------------------------------|
| Python         | 3.12          | 运行时环境                                    |
| FastAPI        | >= 0.100.0    | Web 框架，提供 REST API 与 WebSocket           |
| Pydantic V2    | (FastAPI 依赖) | 数据校验与序列化，定义请求/响应模型                     |
| uvicorn        | >= 0.20.0     | ASGI 服务器，运行 FastAPI 应用                   |
| python-multipart| >= 0.0.6     | 文件上传支持 (CSV 导入)                          |

**选型理由**:

- Python 3.12: 使用了 `from __future__ import annotations` 延迟类型求值、dataclass 增强、改进的错误消息和性能优化
- FastAPI: 原生异步支持、自动 OpenAPI 文档、Pydantic V2 集成、依赖注入系统（`get_db_path`、`get_data_dir` 通过 `app.state` 注入）
- Pydantic V2: Rust 内核验证，使用 `ConfigDict(extra="forbid")` 严格模式和 `ConfigDict(from_attributes=True)` 行模型转换
- uvicorn: 轻量级 ASGI 服务器，支持 `--reload` 开发模式

### 1.3 进化算法

| 技术      | 版本要求       | 实际使用情况     | 说明                         |
|---------|------------|------------|----------------------------|
| deap    | >= 1.4.1   | 声明但未实际使用 | requirements.txt 包含但代码中未导入 |

项目使用自定义进化引擎 `core/evolution/engine.py`，原因：

1. StrategyDNA 的嵌套 dataclass 结构（SignalGene + LogicGenes + ExecutionGenes + RiskGenes + TimeframeLayer）与 deap 的个体表示不兼容
2. 自定义引擎支持 MTF 层级结构的交叉/变异
3. 自适应变异强度需要精细控制（根据停滞代数动态调整变异次数和权重）
4. EarlyStopChecker 支持业务相关的停止规则（target_reached / stagnation / decline / max_generations）

```typescript
    proxy: {
      "/api": { target: "http://localhost:8000", changeOrigin: true },
      "/ws":  { target: "ws://localhost:8000", ws: true },
    },
  },
});
```

开发环境通过 Vite 代理将 `/api` 和 `/ws` 转发到 FastAPI 后端，无需处理跨域。路径别名 `@` 映射到 `src/`。

## 3. 数据库技术

### 3.1 SQLite

| 配置项          | 值    | 说明                     |
|--------------|------|------------------------|
| journal_mode | WAL  | Write-Ahead Logging 模式 |
| row_factory  | Row  | 字典式行访问                  |
| 数据库文件       | data/quant.db     | 主数据库 (strategy, backtest_result, dataset_meta, schema_version) |
| 进化数据库       | data/evolution.db | 进化专用 (evolution_task, generation_snapshot, evolution_history) |

**选型理由**:

- 零配置、无服务端进程、Python 标准库内建支持
- WAL 模式允许读操作与写操作并发（EvolutionRunner 写入，API 路由读取）
- 单机桌面应用场景无需重型数据库
- 两个数据库文件隔离进化高频写入与常规查询

### 3.2 Parquet

| 配置项      | 值              | 说明                      |
|----------|----------------|-------------------------|
| engine   | pyarrow        | Pandas 推荐的 Parquet 引擎    |
| 压缩      | Snappy (默认)    | 压缩率和速度的平衡              |
| 文件命名    | {symbol}_{interval}.parquet | 如 BTCUSDT_4h.parquet |
| 列       | OHLCV + 100+ 指标列 | 列式存储对宽表压缩率高         |
| 索引      | DatetimeIndex (UTC) | 时间序列索引             |

## 8. 安全考虑

### 8.1 CORS 配置

当前开发环境允许所有源 (`allow_origins=["*"]`)，生产环境应限制为前端域名。

### 8.2 数据库安全

- SQLite 文件权限应限制为应用用户可读写
- 不存储交易密钥等敏感信息
- 数据库文件位于 data/ 目录，不应暴露到 HTTP 服务

### 8.3 API 安全

- 当前无身份验证机制（单机桌面应用场景）
- 如需部署到公网，应添加认证中间件
- 文件上传接口应限制文件大小和类型
