# 02 - 数据流全景

本文档追踪 MyQuant 系统中三个核心场景的完整数据流，覆盖从数据输入到最终输出的每一步转换过程。

---

## 1. 场景一：数据摄入

数据摄入是将外部行情数据导入系统的过程。系统支持两种数据源：CSV 文件导入和 API 拉取。最终统一存储为 Parquet 格式。

### 1.1 CSV 文件导入流程

```
用户上传 CSV 文件
       |
       v
[api/routes/data.py] POST /api/data/import
       |
       +-- 解析 FormData (file, symbol, interval, mode)
       |
       v
[core/data/csv_importer.py]
       |
       +-- 1. detect_format(path)
       |       检测首行内容:
       |       - 12 列数字 -> BINANCE_OFFICIAL
       |       - 有表头关键字 -> GENERIC_OHLCV
       |
       +-- 2. read_csv(path)
       |       BINANCE_OFFICIAL:
       |         pd.read_csv(header=None, names=BINANCE_COLUMNS)
       |         detect_timestamp_precision(open_time[0])
       |           13位 -> MILLISECOND -> unit="ms"
       |           16位 -> MICROSECOND -> unit="us"
       |         pd.to_datetime(open_time, unit=ms/us, utc=True)
       |       GENERIC_OHLCV:
       |         pd.read_csv() + 列名重命名 (GENERIC_COLUMN_MAP)
       |         pd.to_datetime(timestamp, utc=True)
       |       统一处理:
       |         set_index("timestamp")
       |         仅保留 [open, high, low, close, volume]
       |         to_numeric + 去重 + 排序
       |
       +-- 3. validate_ohlcv(df)
       |       - NaN 检查
       |       - high >= max(open, close, low) 向量化验证
       |       - low <= min(open, close, high) 向量化验证
       |       - volume >= 0
       |       错误则抛出 ValueError
       |
       +-- 4. parse_filename(path)
       |       正则匹配: {SYMBOL}-{INTERVAL}-{YYYY}-{MM}.csv
       |       示例: BTCUSDT-4h-2025-01.csv -> ("BTCUSDT", "4h")
       |
       +-- 5. 写入 Parquet
               mode=REPLACE:
                 save_parquet(df, parquet_path) -> df.to_parquet(engine="pyarrow")
               mode=MERGE:
                 merge_parquet(df, parquet_path)
                   existing = load_parquet(path)
                   merged = pd.concat([existing, new_df])
                   merged = 去重(keep="last") + 排序
                   save_parquet(merged, path)
               mode=NEW:
                 save_parquet(df, parquet_path)
       |
       v
文件写入: data/market/{symbol}_{interval}.parquet
       |
       v
[api/db_ext.py] save_dataset_meta()
       |
       +-- 写入 dataset_meta 表
           dataset_id = "{symbol}_{interval}"
           记录: row_count, time_start, time_end, file_size_bytes,
                  format_detected, timestamp_precision, quality_status
       |
       v
返回 DataImportResponse (JSON)
```

### 1.2 多文件批量导入

```
多个 CSV 文件
       |
       v
[core/data/csv_importer.py] import_csv_batch()
       |
       +-- 逐文件 read_csv() + validate_ohlcv()
       +-- pd.concat(all_dfs)
       +-- 去重(keep="first") + 排序
       +-- 写入单个 Parquet 文件
       |
       v
返回 CsvImportResult (含 files_processed, rows_imported, time_range)
```

### 1.3 Parquet 文件内部结构

```
Parquet 文件: BTCUSDT_4h.parquet
+-----------------------------+----------+----------+----------+----------+----------+
| timestamp (DatetimeIndex)   | open     | high     | low      | close    | volume   |
+-----------------------------+----------+----------+----------+----------+----------+
| 2024-01-01 00:00:00+00:00   | 42000.5  | 42500.0  | 41800.2  | 42350.0  | 1234.5   |
| 2024-01-01 04:00:00+00:00   | 42350.0  | 42800.5  | 42200.0  | 42600.3  | 987.2    |
+-----------------------------+----------+----------+----------+----------+----------+

引擎: PyArrow
压缩: Snappy (默认)
索引: DatetimeIndex (UTC)
```

## 2. 场景二：策略进化

策略进化是系统的核心流程，从创建进化任务开始，到产出冠军策略 DNA 结束。

### 2.1 任务创建

```
前端 Evolution.tsx
  用户填写: symbol, timeframe, target_score, score_template,
            population_size, max_generations, leverage, direction,
            data_start, data_end, timeframe_pool, mode
  可选: 手动编辑初始 DNA (SeedConfigForm)
       |
       v
POST /api/evolution/tasks
  Body: EvolutionTaskCreate
       |
       v
[api/routes/evolution.py]
       |
       +-- 解析 initial_dna (可选，无则自动生成)
       +-- 生成 task_id (UUID)
       +-- 写入 evolution_task 表 (status='pending')
       +-- 写入 dataset 约束条件
       |
       v
返回 EvolutionTaskResponse (status='pending')
```

### 2.2 后台执行 -- EvolutionRunner 主循环

```
[api/runner.py] EvolutionRunner.run()
       |
       +-- while not stop_event:
             _tick()
               |
               +-- 1. 查询 evolution_task WHERE status='pending'
               |      ORDER BY created_at ASC LIMIT 1
               |
               +-- 2. _run_task(task_row)
                      |
                      +-- update_task(status='running')
                      +-- 解析 initial_dna: StrategyDNA.from_json()
                      +-- 构建 EvolutionEngine
                      |
                      +-- engine.evolve(
                      |      ancestor=dna,
                      |      evaluate_fn=evaluate_fn,
                      |      on_generation=on_generation_callback,
                      |    )
```

### 2.3 进化引擎内部流程

```
[core/evolution/engine.py] EvolutionEngine.evolve()
       |
       v
  === 第 0 步: 初始化种群 ===
       |
       +-- init_population(population_size, ancestor)
       |     |
       |     +-- population[0] = ancestor (或 create_random_dna)
       |     +-- 循环生成剩余个体:
       |           随机选择已有个体作为父代
       |           随机选择变异算子 (mutate_params/indicator/logic/risk)
       |           验证子代合法性 (validate_dna)
       |           合法则加入种群，否则重新生成随机 DNA
       |
       v
  === 第 1-N 代循环 ===
       |
  [循环开始]
       |
       +-- 强制覆盖 task 级约束 (leverage, direction)
       |
       +-- 评估所有个体:
       |     for individual in population:
       |       score = evaluate_fn(individual)  --> 详见 2.4 节
       |
       +-- 排序: scored = sorted(zip(population, scores), by score DESC)
       |
       +-- 计算统计: best_score, avg_score
       |
       +-- 追踪 champion: 如果 best_score > champion_score 则更新
       |
       +-- 触发回调: on_generation(gen, best_score, avg_score)
       |     --> 详见 2.5 节
       |
       +-- 早停检查: EarlyStopChecker.check()
       |     4 条规则:
       |     1. target_reached: best_score >= target 且 gen >= min_generations
       |     2. stagnation: 连续 patience 代无改善 (>= 0.5 分)
       |     3. decline: 连续 decline_limit 代衰退
       |     4. max_generations: 达到最大代数
       |
       +-- 精英保留:
       |     elite_count = max(2, population_size * elite_ratio)
       |     elites = scored[:elite_count]
       |
       +-- 交叉 + 变异:
       |     while children < population_size - elite_count - 2:
       |       p1, p2 = random.sample(elites, 2)
       |       child = crossover(p1, p2)
       |         entry 基因取自 p1, exit 基因取自 p2
       |         logic/risk 随机取自一方
       |         MTF 层: 对应位置交叉
       |       n_mutations = 自适应选择 (1-4 次)
       |         停滞 > 8 代: weights=[2,3,4] -> 30/45/25
       |         停滞 > 4 代: weights=[1,2,3] -> 25/45/30
       |         正常:        weights=[1,2,3] -> 50/35/15
       |       for _ in range(n_mutations):
       |         mut = random.choice(mutation_pool)
       |         child = mut(child)
       |
       +-- 注入新鲜血液:
       |     inject_fresh_blood(population, n=random(1,2))
       |
       +-- 多样性维护:
       |     check_and_maintain_diversity(population)
       |       计算个体 DNA 指纹 (indicator+params+role 哈希)
       |       替换克隆个体为随机 DNA
       |
       +-- 截断到 population_size
       |
       +-- 回到 [循环开始]
       |
       v
  === 返回结果 ===
       |
       +-- return {
              champion: best_dna,
              champion_score: best_score,
              history: [...],
              stop_reason: "target_reached" | "stagnation" | "decline" | "max_generations",
              total_generations: N,
            }
```

### 2.4 单个体评估流程 (evaluate_fn)

这是进化过程中每个个体被调用的核心评估路径，将 DNA 转化为评分。

```
StrategyDNA 个体
       |
       v
[api/runner.py] EvolutionRunner._evaluate_dna()
       |
       +-- 强制覆盖 leverage 和 direction
       |
       v
[core/data/mtf_loader.py] load_and_prepare_df()
       |
       +-- find_parquet(data_dir, symbol, timeframe)
       |     尝试: {symbol}_{timeframe}.parquet
       |     尝试: 别名 (60m->1h, 1D->1d 等)
       |
       +-- load_parquet(path) -> pd.read_parquet(engine="pyarrow")
       |
       +-- 日期切片:
       |     if data_start: df = df[df.index >= data_start]
       |     if data_end:   df = df[df.index <= data_end]
       |
       +-- compute_all_indicators(df)   <--- [core/features/indicators.py]
       |     |
       |     +-- 遍历 _DEFAULT_PARAMS 中 30 个指标的 80+ 种参数组合
       |     +-- _compute_indicator(df, name, params) 对每个指标:
       |           EMA:    ta.ema(close, length=period) -> ema_{period}
       |           RSI:    ta.rsi(close, length=period) -> rsi_{period}
       |           MACD:   ta.macd(close, fast, slow, signal) -> 3 列
       |           BB:     ta.bbands(close, length, std) -> 3 列 (upper/middle/lower)
       |           ... (30 种指标，每种 1-5 种参数)
       |     +-- 返回增强 DataFrame (OHLCV + 100+ 指标列)
       |
       v
enhanced_df (100+ 列的 DataFrame)
       |
       v
[可选] MTF 数据加载: load_mtf_data()
       |
       +-- 对 timeframe_pool 中每个额外时间框架:
             find_parquet() -> load_parquet() -> 切片 -> compute_all_indicators()
       +-- 返回 {timeframe: enhanced_df} 字典
       |
       v
[core/strategy/executor.py] dna_to_signals(dna, enhanced_df, dfs_by_timeframe)
       |
       +-- MTF 模式:
       |     for layer in dna.layers:
       |       layer_df = dfs_by_timeframe[layer.timeframe]
       |       evaluate_layer(layer, layer_df)
       |         for gene in layer.signal_genes:
       |           indicator_col = _get_indicator_column(df, gene)
       |             根据 indicator 名称和 params 构建列名
       |             精确匹配 -> 前缀匹配 -> 抛出 ValueError
       |           signal = evaluate_condition(indicator_col, close, gene.condition, df)
       |             根据 condition.type 分派到对应逻辑:
       |             lt/gt/le/ge:      简单比较
       |             cross_above/below: shift(1) 比较
       |             price_above/below: close vs indicator
       |             cross_above/below_series: 两个指标交叉
       |             lookback_any/all: rolling window + inner condition
       |             touch_bounce:     近似度 + 反弹
       |             role_reversal:    支撑/阻力角色切换
       |             wick_touch:       影线触碰
       |           按 role 分类: entry_trigger/entry_guard/exit_trigger/exit_guard
       |         combine_signals(all_entry, entry_logic)  # AND/OR
       |         combine_signals(all_exit, exit_logic)
       |       resample_signals() 如果跨时间框架
       |     combine_signals(layer_entries, cross_layer_logic)
       |     去重: entries & ~both
       |
       +-- 单时间框架模式:
             同上但不处理 layers，直接使用 dna.signal_genes
       |
       v
entries: pd.Series[bool], exits: pd.Series[bool]
       |
       v
[core/backtest/engine.py] BacktestEngine.run(dna, enhanced_df, dfs_by_timeframe)
       |
       +-- dna_to_signals()  (同上)
       +-- 构建 vectorbt Portfolio:
       |     vbt.Portfolio.from_signals(
       |       close, entries, exits, open, high, low,
       |       init_cash, fees, slippage,
       |       sl_stop=stop_loss, tp_stop=take_profit,
       |       size=position_size * leverage,
       |       size_type="percent", accumulate=False,
       |       direction=0(long)/1(short),
       |     )
       |
       +-- equity_curve = portfolio.value()
       +-- _apply_funding_costs(equity_curve, leverage, timeframe)
       |     每 8 小时扣除资金费率: RATE_PER_8H * periods_per_bar * borrowed_ratio
       +-- _check_liquidation(equity_curve, leverage, init_cash)
       |     维持保证金: init_cash * (1 - 0.9/leverage)
       |     如果跌破则强制清零
       +-- 提取交易数据:
             total_trades = portfolio.trades.count()
             trade_pnl -> trade_win_rate, trade_returns
       |
       v
[core/scoring/metrics.py] compute_metrics(equity_curve, total_trades, ...)
       |
       +-- annual_return: (end/start)^(1/years) - 1
       +-- max_drawdown:  min((equity - cummax) / cummax)
       +-- sharpe_ratio:
       |     优先使用交易级别: (mean(trade_returns) / std(trade_returns)) * sqrt(trades/year)
       |     回退使用 bar 级别: (mean(bar_returns) / std(bar_returns)) * sqrt(bars/year)
       +-- win_rate: 优先使用交易级别胜率，回退到 bar 级别
       +-- calmar_ratio: |annual_return / max_drawdown|
       |
       v
metrics = {annual_return, sharpe_ratio, max_drawdown, win_rate, calmar_ratio, total_trades}
       |
       v
[core/scoring/scorer.py] score_strategy(metrics, template_name)
       |
       +-- 获取评分模板 (profit_first/steady/risk_first/custom)
       +-- total_trades == 0 -> score = 0
       +-- 归一化每个维度到 0-100:
       |     annual_return: [-1.0, 1.0] -> [0, 100]
       |     sharpe_ratio:  [0, 3.0] -> [0, 100]
       |     max_drawdown:  [0%, 50%] -> [100, 0] (二次惩罚)
       |     win_rate:      [40%, 70%] -> [0, 100]
       |     calmar_ratio:  [0, 5.0] -> [0, 100]
       +-- 加权求和:
       |     total = sum(normalized[dim] * weight[dim])
       +-- 交易次数惩罚:
       |     trade_factor = min(1.0, total_trades / 10)
       |     total *= trade_factor
       |
       v
score = total_score (0-100 浮点数)
```

```
       +-- 更新 evolution_task.current_generation (SQLite)
```

## 3. 场景三：假设验证

假设验证允许用户表达 WHEN/THEN 条件，系统在历史数据上统计验证。

### 3.1 完整流程

```
用户在 Lab.tsx 页面构建 WHEN/THEN 条件
       |
       WHEN: { subject: indicator, action: condition_type, target: threshold/indicator }
       THEN: { subject: price_change, action: gt/lt, target: percentage }
       |
       v
POST /api/validate
  Body: {
    pair, timeframe, dataset_id,
    when_conditions: [...],
    then_condition: {...},
    look_ahead_bars: 20,
    data_start, data_end
  }
       |
       v
[api/routes/validate.py]
       |
       v
[core/validation/engine.py] validate_hypothesis()
       |
       +-- 1. 加载数据:
       |     load_parquet(parquet_path) -> raw OHLCV DataFrame
       |     compute_all_indicators(df) -> enhanced DataFrame
       |
       +-- 2. 评估 WHEN 触发器:
       |     for each bar in enhanced_df:
       |       evaluate when_conditions
       |       combine with AND logic
       |       if triggered -> 记录 TriggerRecord
       |
       +-- 3. Look-ahead THEN 评估:
       |     for each trigger point:
       |       计算后续 look_ahead_bars 内的价格变化
       |       判断 THEN 条件是否满足
       |       matched = then_condition_satisfied
       |
       +-- 4. 统计分析:
       |     match_rate = match_count / total_count
       |     distribution: 变化百分比的直方图分桶
       |     percentiles: p10, p25, p50, p75, p90
       |     concentration: 正/负/中性分布
       |     extremes: 最大正/负变化案例
       |
       +-- 5. 模式检测 (可选):
       |     [core/validation/patterns.py]
       |     检测价格形态 (支撑/阻力突破、趋势延续/反转等)
       |
       v
ValidationResult {
  match_rate, total_count, match_count, mismatch_count,
  triggers: List[TriggerRecord],
  distribution: List[DistributionBucket],
  percentiles, concentration, extremes,
  warnings
}
       |
       v
返回 JSON -> 前端展示
       |
       v
[web/src/pages/Lab.tsx]
  +-- TriggerTable: 显示触发记录
  +-- DistributionChart: 分布直方图
  +-- ValidationConclusion: 验证结论摘要
```

### 3.2 触发器评估细节

```
enhanced_df 中的每一行 bar:
       |
       v
评估 WHEN 条件组:
  例: WHEN RSI_14 < 30 AND close > ema_50
       |
       +-- 提取指标列:
       |     rsi_14 = enhanced_df["rsi_14"]
       |     ema_50 = enhanced_df["ema_50"]
       |     close  = enhanced_df["close"]
       |
       +-- 评估每个条件:
       |     cond1 = rsi_14 < 30          -> bool Series
       |     cond2 = close > ema_50       -> bool Series
       |
       +-- 组合: triggered = cond1 & cond2
       |
       v
对于每个 triggered=True 的 bar:
       |
       +-- 记录: {id, time, trigger_price, indicator_values}
       |
       +-- Look-ahead:
             从当前 bar 向后看 look_ahead_bars (如 20 根)
             计算 price_change = (future_close - current_close) / current_close
             判断 THEN: price_change > 2%?
             matched = True/False
```

```
    |     [backtest/engine] BacktestEngine.run() -> BacktestResult
    |         |
    |         v
    |     [scoring/metrics] compute_metrics() -> metrics dict
    |         |
    |         v
    |     [scoring/scorer] score_strategy() -> total_score (0-100)
    |
    +---> [validation/engine] validate_hypothesis() -> ValidationResult
```

### 4.2 数据格式在各模块间的流转

## 5. 异常路径与容错

### 5.1 数据不可用

当 Parquet 文件不存在或数据不足时：

```
load_and_prepare_df() 返回 None
       |
       v
_evaluate_dna() 检测到 None
       |
       +-- diagnostics["fallback"] = True
       +-- score = random.uniform(10, 50)  (随机低分，保持进化继续)
       |
       v
该个体获得低分，大概率在下一代被淘汰
```

### 5.2 零交易策略

```
dna_to_signals() -> entries 全为 False
       |
       v
BacktestEngine -> total_trades = 0
       |
       v
compute_metrics() -> 所有指标为 0
       |
       v
score_strategy() -> total_trades == 0 -> score = 0.0
```

### 5.3 杠杆清算

```
BacktestEngine.run()
       |
       +-- _check_liquidation(equity_curve, leverage, init_cash)
       |     如果 equity < init_cash * (1 - 0.9/leverage):
       |       equity_curve 截断为 0
       |       liquidated = True
       |
       v
equity_curve 被清零 -> total_return 极低 -> score 极低
```

### 5.4 进化任务中途停止

```
用户点击 "Stop" -> PATCH /api/evolution/tasks/{id} {action: "stop"}
       |
       v
evolution_task.status -> "stopped"
       |
       v
on_generation 回调中检测到:
  t = get_task(task_id)
  t["status"] not in ("running", "pending")
       |
       v
抛出 _StopEvolution 异常
       |
       v
engine.evolve() 被中断
update_task(status="stopped")
```
