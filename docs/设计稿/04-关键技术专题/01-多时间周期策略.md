# 01 - 多时间周期策略 (Multi-Timeframe Strategy)

## 1 概述

多时间周期策略(Multi-Timeframe, MTF)是 MyQuant 系统中最复杂的架构特性。其核心思想是:将多个不同粒度的K线数据组合使用,在更高时间周期上判断趋势方向,在更低时间周期上精确执行买卖操作,从而提升策略的稳定性和收益质量。

本文档描述 MTF 的完整架构设计,包括配置流程、DNA 数据结构、种群初始化、遗传算子、信号生成和数据加载六个核心环节。

---

## 2 设计原则

**时间周期组合是任务级硬约束,不是可进化对象。**

用户在创建进化任务时选定一组有序时间周期列表(如 ["3d", "4h", "15m"]),该列表在整个进化过程中保持不变。遗传算法只能优化每个固定层级内的信号参数(指标选择、参数值、条件类型、逻辑组合),不能改变层级数量、层级顺序或各层级对应的时间周期。

这一设计决策基于以下理由:

- 时间周期组合是人类的宏观市场认知决策,属于"策略架构"层面
- 遗传算法擅长微观参数搜索,不擅长结构性的组合优化
- 固定层级结构使交叉操作有明确的对应关系,避免无效重组
- 减少搜索空间维度,加速收敛

## 3 配置流程

### 3.1 用户界面层

用户在 AutoConfigForm(前端配置表单)中选择一个有序的时间周期列表。典型选择示例:

- ["3d", "4h", "15m"] -- 三层结构:月级趋势 / 日级确认 / 小时级执行
- ["1d", "4h"] -- 两层结构:日级趋势 / 小时级执行
- ["4h"] -- 单层结构:无 MTF,等同于传统单时间周期策略

### 3.2 前端处理

前端对用户选择的时间周期列表进行排序处理:

1. 按持续时间从长到短排序(最长优先)
2. 将排序后的列表作为 `timeframe_pool` 字段发送至后端

排序规则示例:
- 用户选择 ["15m", "3d", "4h"]
- 排序后变为 ["3d", "4h", "15m"]

排序依据为各时间周期对应的分钟数:

| 时间周期 | 分钟数 |
|---------|-------|
| 1m      | 1     |
| 5m      | 5     |
| 15m     | 15    |
| 30m     | 30    |
| 1h      | 60    |
| 4h      | 240   |
| 1d      | 1440  |
| 3d      | 4320  |

时间周期到分钟数的转换逻辑位于 `core/validation/mtf.py` 的 `_TF_MINUTES` 字典和 `get_timeframe_minutes()` 函数中。

### 3.3 后端推导

后端收到 `timeframe_pool` 后自动推导执行时间周期:

- 执行时间周期(`execution_timeframe`) = `timeframe_pool` 中最短的时间周期
- 所有非执行时间周期自动成为 DNA 中的层级(layer)

推导示例:

```
timeframe_pool = ["3d", "4h", "15m"]

推导结果:
  execution_timeframe = "15m"
  layers[0] -> "3d" (趋势层)
  layers[1] -> "4h" (确认层)
  signal_genes -> 在 "15m" 上执行(执行层)
```

### 3.4 任务创建参数

API 层通过 `EvolutionTaskCreate` schema 接收以下 MTF 相关字段:

```python
class EvolutionTaskCreate(BaseModel):
    timeframe_pool: Optional[List[str]] = None  # 有序时间周期列表
    timeframe: str                               # 主时间周期(兼容旧接口)
    leverage: int = 1                            # 杠杆约束
    direction: str = "long"                      # 方向约束
```

`runner.py` 在 `_run_task()` 中解析 `timeframe_pool`,并传递给 `EvolutionEngine`:

```python
tf_pool_raw = task_row.get("timeframe_pool")
tf_pool = json.loads(tf_pool_raw) if isinstance(tf_pool_raw, str) else tf_pool_raw

engine = EvolutionEngine(
    timeframe_pool=tf_pool,
    ...
)
```

## 4 DNA 数据结构

### 4.1 核心数据类

MTF 策略的 DNA 由以下数据类构成(定义于 `core/strategy/dna.py`):

```
StrategyDNA
  |-- signal_genes: List[SignalGene]      # 执行时间周期的信号基因
  |-- logic_genes: LogicGenes              # 执行时间周期的逻辑基因
  |-- execution_genes: ExecutionGenes      # 包含 timeframe, symbol
  |-- risk_genes: RiskGenes               # 风控参数
  |-- layers: Optional[List[TimeframeLayer]]  # MTF 层级列表
  |-- cross_layer_logic: str              # 跨层级组合逻辑 ("AND" / "OR")
  |-- _layers_explicit: bool              # 标记:层级是否为显式创建
```

### 4.2 TimeframeLayer 结构

每个 MTF 层级包含:

```
TimeframeLayer
  |-- timeframe: str                       # 该层级对应的时间周期
  |-- signal_genes: List[SignalGene]       # 该层级的信号基因
  |-- logic_genes: LogicGenes              # 该层级的逻辑基因
```

每个 `SignalGene` 包含:

```
SignalGene
  |-- indicator: str          # 指标名称(如 "RSI", "EMA")
  |-- params: Dict            # 指标参数(如 {"period": 14})
  |-- role: SignalRole        # 信号角色(entry_trigger/entry_guard/exit_trigger/exit_guard)
  |-- field_name: Optional[str]  # 多输出指标的字段选择(如 MACD 的 "histogram")
  |-- condition: Dict         # 条件字典(如 {"type": "lt", "threshold": 30})
```

### 4.3 三层示例

以 `timeframe_pool = ["3d", "4h", "15m"]` 为例:

```python
StrategyDNA(
    execution_genes=ExecutionGenes(timeframe="15m", symbol="BTCUSDT"),
    signal_genes=[...],      # 在 15m 上评估的 entry/exit 信号
    logic_genes=LogicGenes(entry_logic="AND", exit_logic="OR"),
    layers=[
        TimeframeLayer(
            timeframe="3d",  # 趋势层
            signal_genes=[...],
            logic_genes=LogicGenes(...),
        ),
        TimeframeLayer(
            timeframe="4h",  # 确认层
            signal_genes=[...],
            logic_genes=LogicGenes(...),
        ),
    ],
    cross_layer_logic="AND",  # 三层信号全部通过才触发交易
)
```

信号评估语义:当 3d 层信号、4h 层信号、15m 层信号全部满足(cross_layer_logic="AND")时,触发入场/出场操作。

### 4.4 is_mtf 属性

`StrategyDNA.is_mtf` 属性用于判断 DNA 是否使用多时间周期:

```python
@property
def is_mtf(self) -> bool:
    return self.layers is not None and len(self.layers) >= 1 and self._layers_explicit
```

判断条件:
- `layers` 不为 None
- 至少包含 1 个层级
- `_layers_explicit` 为 True

**向后兼容**:当旧格式 DNA 只有 `signal_genes` 没有 `layers` 时,`from_dict()` 会自动将 `signal_genes` 包装为单层 `TimeframeLayer`,但此时 `_layers_explicit` 为 False,因此 `is_mtf` 返回 False。这确保了自动包装的单层 DNA 不会被误判为 MTF 策略。

### 4.5 序列化与反序列化

DNA 通过 `to_dict()` / `from_dict()` 方法实现序列化:

- `to_dict()`: 将 `layers` 序列化为字典列表,包含完整层级信息
- `from_dict()`: 解析 `layers` 字段,重建 `TimeframeLayer` 对象;若存在 `layers`,设置 `_layers_explicit = True`
- 自动包装:若 `layers` 为 None 但 `signal_genes` 非空,自动创建单层包装(此时 `_layers_explicit` 保持 False)

## 5 种群初始化

### 5.1 create_random_dna()

位于 `core/evolution/population.py`。

当传入 `timeframe_pool` 且长度大于 1 时,100% 为所有非执行时间周期创建 MTF 层级:

```python
def create_random_dna(
    timeframe: str = "4h",
    symbol: str = "BTCUSDT",
    timeframe_pool: Optional[List[str]] = None,
    leverage: int = 1,
    direction: str = "long",
) -> StrategyDNA:
    # ... 创建基础 DNA(执行层的信号基因)...

    # 为所有非执行时间周期创建层级
    if timeframe_pool and len(timeframe_pool) > 1:
        other_tfs = [tf for tf in timeframe_pool if tf != timeframe]
        if other_tfs:
            layers = [create_random_mtf_layer(tf, symbol) for tf in other_tfs]
            dna.layers = layers
            dna._layers_explicit = True
            dna.cross_layer_logic = random.choice(["AND", "OR"])

    return dna
```

关键行为:
- `timeframe` 参数即为执行时间周期(最短的时间周期)
- `other_tfs` 排除执行时间周期后,按列表顺序为每个剩余时间周期创建一个层级
- 每个层级通过 `create_random_mtf_layer()` 独立生成

### 5.2 create_random_mtf_layer()

```python
def create_random_mtf_layer(
    timeframe: str,
    symbol: str = "BTCUSDT",
) -> TimeframeLayer:
```

生成逻辑:
1. 从所有非 guard_only 指标中随机选择,创建 1 个 `ENTRY_TRIGGER`
2. 50% 概率添加 1 个 `ENTRY_GUARD`
3. 从所有非 guard_only 指标中随机选择,创建 1 个 `EXIT_TRIGGER`
4. 随机选择 entry_logic 和 exit_logic(AND/OR)

每个信号基因的参数从指标注册表(`INDICATOR_REGISTRY`)的参数定义范围内随机生成,条件类型从该指标支持的 `supported_conditions` 列表中随机选取。

### 5.3 init_population()

`init_population()` 在初始化种群时,将 `timeframe_pool` 传递给 `create_random_dna()`:

```python
def init_population(
    size: int = 15,
    ancestor: Optional[StrategyDNA] = None,
    timeframe_pool: Optional[List[str]] = None,
    ...
) -> List[StrategyDNA]:
    # 第一个个体:使用 ancestor 或创建随机 DNA
    if ancestor is not None:
        population.append(ancestor)
    else:
        population.append(create_random_dna(
            timeframe, symbol,
            leverage=leverage, direction=direction,
            timeframe_pool=timeframe_pool,
        ))

    # 后续个体:通过变异祖先生成
    while len(population) < size:
        parent = random.choice(population[:min(len(population), 5)])
        child = random.choice(mutation_funcs)(parent)
        ...
```

## 6 遗传算子

### 6.1 交叉操作 - crossover()

位于 `core/evolution/operators.py`。

交叉操作的核心约束:**保持层级结构不变,只交叉对应层级的信号基因**。

执行流程:

1. **执行层(base signal_genes)交叉**: 从父本 A 取 entry 类信号(entry_trigger + entry_guard),从父本 B 取 exit 类信号(exit_trigger + exit_guard)
2. **MTF 层级交叉**: 当两个父本都有 layers 时,按位置一一配对:
   - 对每对 `layer_A[i]` 和 `layer_B[i]`:
     - 从 layer_A 取 entry 信号
     - 从 layer_B 取 exit 信号
     - logic_genes 随机选择 A 或 B
     - 保持 timeframe 不变(使用 layer_A 的 timeframe)
3. **cross_layer_logic**: 随机选择父本 A 或 B 的值
4. **风险基因**: 随机选择父本 A 或 B

```
父本 A:                               父本 B:
  signal_genes: [entry_A, exit_A]       signal_genes: [entry_B, exit_B]
  layers[0]("3d"): [L0_entry_A, L0_exit_A]
                                        layers[0]("3d"): [L0_entry_B, L0_exit_B]
  layers[1]("4h"): [L1_entry_A, L1_exit_A]
                                        layers[1]("4h"): [L1_entry_B, L1_exit_B]

子代:
  signal_genes: [entry_A, exit_B]       <- 执行层: entry 来自 A, exit 来自 B
  layers[0]("3d"): [L0_entry_A, L0_exit_B]
  layers[1]("4h"): [L1_entry_A, L1_exit_B]
```

### 6.2 _pick_signal_pool()

该函数决定变异操作作用于哪一层信号基因池:

```python
def _pick_signal_pool(data: dict) -> dict:
    layers = data.get("layers")
    if not layers:
        return {"genes": data["signal_genes"]}

    n_targets = 1 + len(layers)  # base + 每个层级
    choice = random.randint(0, n_targets - 1)
    if choice == 0:
        return {"genes": data["signal_genes"]}  # base 层
    else:
        layer_idx = choice - 1
        return {"genes": layers[layer_idx]["signal_genes"]}  # 某个层级
```

等概率选择策略:base 层和每个 MTF 层级享有相同的被选中概率。例如三层级 MTF(1 base + 2 layers),每个被选中的概率为 1/3。

### 6.3 参数变异 - mutate_params()

通过 `_pick_signal_pool()` 随机选择目标层级后:
1. 从该层级的信号基因中随机选择一个有参数的基因
2. 从该指标的参数定义中随机选择一个参数
3. 在当前值基础上施加随机扰动(delta = +/- (5%~30%) * range)
4. 按参数步长对齐并钳位到合法范围

### 6.4 指标替换 - mutate_indicator()

通过 `_pick_signal_pool()` 选择目标层级后:
1. 随机选择该层级中的一个信号基因
2. 从同类别指标中选取替代指标(通过 `get_interchangeable()`)
3. 替换指标名称,重新生成参数和条件

### 6.5 逻辑变异 - mutate_logic()

从所有可用的逻辑基因集合(base + 各层级)中随机选择一个:
- 可选动作: flip_entry / flip_exit / flip_both
- 翻转 AND <-> OR

### 6.6 跨层级逻辑变异 - mutate_cross_logic()

仅当 `dna.is_mtf` 为 True 时生效:

```python
def mutate_cross_logic(dna: StrategyDNA) -> StrategyDNA:
    if not dna.is_mtf:
        return dna
    # 翻转 cross_layer_logic: AND <-> OR
```

此操作在进化引擎的变异池中仅当 `timeframe_pool` 长度大于 1 时才被加入:

```python
if len(self.timeframe_pool) > 1:
    mutation_pool.append(mutate_cross_logic)
```

### 6.7 添加/删除信号 - mutate_add_signal() / mutate_remove_signal()

通过 `_pick_signal_pool()` 选择目标层级后:
- `mutate_add_signal()`: 在选定的层级中添加一个 guard 信号
- `mutate_remove_signal()`: 从选定的层级中移除一个 guard 信号

### 6.8 已移除的操作

以下操作在当前的"硬约束"设计下已不再使用:

| 操作 | 原功能 | 移除原因 |
|------|--------|---------|
| mutate_add_layer | 添加新的时间周期层级 | 层级数量由用户固定,不可增加 |
| mutate_remove_layer | 移除现有层级 | 层级数量由用户固定,不可减少 |
| mutate_layer_timeframe | 修改层级的时间周期 | 时间周期分配由用户固定,不可更改 |

这三个函数虽然在 `operators.py` 中保留代码定义,但不再被进化引擎的变异池调用。

## 7 信号生成

### 7.1 dna_to_signals()

位于 `core/strategy/executor.py`。

这是将 DNA 转化为布尔信号序列的核心函数。MTF 路径的执行流程:

```
输入: dna(StrategyDNA), enhanced_df(DataFrame), dfs_by_timeframe(Dict[str, DataFrame])
输出: (entries: Series, exits: Series)
```

执行步骤:

1. 检查 `dna.is_mtf` 且 `dfs_by_timeframe` 不为 None,进入 MTF 路径
2. 对每个 layer:
   a. 从 `dfs_by_timeframe` 中获取该层级时间周期对应的 DataFrame
   b. 调用 `evaluate_layer(layer, layer_df)` 计算该层级的 entry/exit 信号
   c. 若层级时间周期 != 执行时间周期,调用 `resample_signals()` 将信号对齐到执行时间周期
   d. 若层级时间周期 == 执行时间周期,直接 reindex 到 `enhanced_df` 的索引
3. 使用 `combine_signals()` 按 `cross_layer_logic` 组合所有层级的信号
4. 处理冲突:若某时刻同时为 entry 和 exit,优先 exit(entry = entry & ~both)

### 7.2 evaluate_layer()

```python
def evaluate_layer(layer, df: pd.DataFrame) -> tuple[pd.Series, pd.Series]:
```

对单个 TimeframeLayer 执行信号评估:

1. 遍历该层级的所有 signal_genes
2. 对每个基因:
   a. 通过 `_get_indicator_column()` 从 DataFrame 中解析出指标列
   b. 调用 `evaluate_condition()` 计算布尔信号
   c. 根据 role 分类收集:entry_triggers / entry_guards / exit_triggers / exit_guards
3. 使用该层级的 `logic_genes.entry_logic` 组合所有 entry 信号
4. 使用该层级的 `logic_genes.exit_logic` 组合所有 exit 信号

### 7.3 resample_signals()

```python
def resample_signals(
    high_tf_signals: pd.Series,
    target_index: pd.DatetimeIndex,
) -> pd.Series:
```

将高时间周期信号前向填充(forward-fill)到执行时间周期的索引:

```python
reindexed = high_tf_signals.reindex(target_index, method="ffill")
return reindexed.fillna(False).astype(bool)
```

核心原理:
- 高时间周期(如 3d)的信号产生频率远低于执行时间周期(如 15m)
- 一个 3d 级别的信号在其生效的 3 天内对所有 15m K线持续有效
- `reindex(method="ffill")` 实现了这一"信号持续"语义
- 对齐前无法填充的位置(信号产生前的时段)默认为 False

### 7.4 信号组合示例

以 `timeframe_pool = ["3d", "4h", "15m"]`, `cross_layer_logic = "AND"` 为例:

```
时刻 T (15m K线):

3d 层信号: evaluate_layer(3d_layer, df_3d) -> resample_signals(signals, 15m_index)
  -> 3d_entry[T] = True (3d 级别判断为上涨趋势)

4h 层信号: evaluate_layer(4h_layer, df_4h) -> resample_signals(signals, 15m_index)
  -> 4h_entry[T] = True (4h 级别确认买入信号)

15m 执行层: 直接使用 enhanced_df 评估
  -> 15m_entry[T] = True (15m 级别出现买入触发)

最终 entry[T] = 3d_entry[T] AND 4h_entry[T] AND 15m_entry[T] = True
```

## 8 数据加载

### 8.1 mtf_loader.py

位于 `core/data/mtf_loader.py`,提供两个核心函数:

#### load_and_prepare_df()

加载单个时间周期的数据并计算所有指标:

```
输入: data_dir, symbol, timeframe, data_start, data_end
处理: 查找 parquet 文件 -> 加载 -> 按日期切片 -> compute_all_indicators()
输出: DataFrame (OHLCV + 指标列) 或 None
```

文件查找逻辑:
1. 尝试精确匹配: `{symbol}_{timeframe}.parquet`
2. 尝试别名匹配: 查找 `_TF_ALIASES` 中的替代名称

别名映射:
```python
_TF_ALIASES = {
    "1h": ["1h", "60m"],
    "4h": ["4h"],
    "1d": ["1d", "1D"],
    ...
}
```

#### load_mtf_data()

加载多个时间周期的数据:

```
输入: data_dir, symbol, exec_timeframe, enhanced_df, needed_tfs, data_start, data_end
处理: 遍历 needed_tfs, 加载并计算指标
输出: Dict[str, DataFrame] 或 None
```

返回字典格式: `{timeframe: DataFrame}`,其中始终包含执行时间周期(使用已有的 `enhanced_df`)。

### 8.2 Runner 数据加载

`api/runner.py` 中的 `_evaluate_dna()` 负责 MTF 数据的加载:

```python
def _evaluate_dna(self, individual, task_row, leverage, direction):
    # 1. 加载执行时间周期数据
    enhanced_df = load_and_prepare_df(self.data_dir, symbol, timeframe, ...)

    # 2. 解析 timeframe_pool
    tf_pool = json.loads(task_row.get("timeframe_pool"))

    # 3. 当 timeframe_pool 长度 > 1 时加载 MTF 数据
    if tf_pool and len(tf_pool) > 1:
        dfs_by_timeframe = load_mtf_data(
            self.data_dir, symbol, timeframe, enhanced_df,
            set(tf_pool), data_start, data_end,
        )

    # 4. 信号生成(传入 dfs_by_timeframe)
    entries, exits = dna_to_signals(individual, enhanced_df,
                                    dfs_by_timeframe=dfs_by_timeframe)
```

### 8.3 数据需求与存储格式

每个时间周期的数据以 Parquet 格式独立存储:

```
data/market/
  |-- BTCUSDT_15m.parquet
  |-- BTCUSDT_4h.parquet
  |-- BTCUSDT_1d.parquet
  |-- BTCUSDT_3d.parquet
  |-- ETHUSDT_15m.parquet
  |-- ...
```

每个 Parquet 文件包含:
- DatetimeIndex
- OHLCV 列: open, high, low, close, volume

指标列在加载时通过 `compute_all_indicators()` 动态计算并附加,不存储在 Parquet 中。

## 9 进化引擎集成

### 9.1 EvolutionEngine 的 MTF 支持

`EvolutionEngine` 通过 `timeframe_pool` 参数感知 MTF:

```python
class EvolutionEngine:
    def __init__(self, ..., timeframe_pool: Optional[list] = None):
        self.timeframe_pool = timeframe_pool or []
```

在 `evolve()` 方法中,`timeframe_pool` 被传递到以下环节:
- `init_population()`: 初始种群使用 MTF 层级
- `create_random_dna()`: 新鲜血液注入使用 MTF 层级
- `inject_fresh_blood()`: 多样性维护使用 MTF 层级
- `check_and_maintain_diversity()`: 克隆替换使用 MTF 层级
- 变异池中动态添加 `mutate_cross_logic`

### 9.2 多样性维护与 MTF

`diversity.py` 中的 `inject_fresh_blood()` 和 `check_and_maintain_diversity()` 都接受 `timeframe_pool` 参数:

```python
def inject_fresh_blood(population, n, leverage, direction, timeframe_pool):
    fresh = create_random_dna(
        ...,
        timeframe_pool=timeframe_pool,
    )
    population.append(fresh)
```

这确保了注入的新个体也具有完整的 MTF 层级结构。

## 10 边界情况处理

### 10.1 数据缺失

当某个时间周期的 Parquet 文件不存在时:
- `load_mtf_data()` 跳过该时间周期
- `dfs_by_timeframe` 中不包含该时间周期的键
- `dna_to_signals()` 中对应层级的 DataFrame 查找返回 None,该层级被跳过
- 最终信号仅由可用层级组合而成

风险:当关键层级(如趋势层)数据缺失时,策略可能退化为准单时间周期策略。建议在任务创建时校验数据可用性。

### 10.2 单时间周期回退

当 `timeframe_pool` 长度为 1 或为 None 时:
- `create_random_dna()` 不创建任何层级
- `dna.is_mtf` 返回 False
- `dna_to_signals()` 走单时间周期路径,仅评估 `signal_genes`
- 完全向后兼容非 MTF 策略

### 10.3 信号时间对齐

`resample_signals()` 使用 `reindex(method="ffill")` 进行对齐:
- 高时间周期信号在其下一个信号产生前持续有效
- 数据开头无信号填充的位置默认为 False(无交易)
- 对齐使用 `enhanced_df.index` 作为目标索引,确保与执行时间周期完全一致
