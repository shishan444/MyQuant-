# 03 - 信号条件类型设计 (Signal Condition Types)

## 1 概述

信号条件类型是策略 DNA 中最核心的表达能力单元。每个信号基因(SignalGene)通过一个条件字典(condition dict)定义"在什么情况下该信号触发",条件类型的多样性直接决定进化算法能探索的策略空间。

本文档描述系统中 15 种已实现的条件类型,按 4 个阶段组织,涵盖定义、实现机制、参数结构和生成逻辑。

---

## 2 条件类型总览

| 阶段 | 类型 | 数量 | 核心能力 |
|------|------|------|---------|
| Phase 1 | 基础阈值/价格比较 | 8 | 指标与固定阈值或价格的比较 |
| Phase 2 | 动态序列/回溯窗口 | 4 | 指标间交叉、时间窗口内条件满足 |
| Phase 3 | 支撑/阻力 | 3 | 价格与指标线的接触/角色互换 |
| Phase 4 | 复杂形态 | 2+ | 背离检测、连续方向(验证引擎实现) |

`ConditionType` 枚举定义于 `core/strategy/dna.py`,列出全部 15 种类型字符串。

---

## 3 Phase 1 - 基础阈值/价格比较 (8 种)

### 3.1 阈值比较 (lt / gt / le / ge)

将指标列当前值与固定阈值比较,使用 pandas 向量化操作:

| 类型 | 公式 | 典型应用 |
|------|------|---------|
| lt | `indicator < threshold` | RSI < 30 (超卖) |
| gt | `indicator > threshold` | RSI > 70 (超买) |
| le | `indicator <= threshold` | ADX <= 20 (无趋势) |
| ge | `indicator >= threshold` | ADX >= 25 (趋势确立) |

条件字典: `{"type": "lt", "threshold": 30}`

随机生成策略: RSI 使用预定义阈值列表 `[25, 30, 35, 40, 60, 65, 70, 75]`,其他指标使用 `uniform(-1, 1)`。

### 3.2 穿越阈值 (cross_above / cross_below)

检测指标从阈值一侧穿越到另一侧的瞬间:

- cross_above: `shift(1) < threshold` 且 `current >= threshold`
- cross_below: `shift(1) > threshold` 且 `current <= threshold`

条件字典: `{"type": "cross_above", "threshold": 0}`

典型应用: MACD histogram 穿越零轴。若 threshold 为 None,返回全 False。

### 3.3 价格比较 (price_above / price_below)

收盘价与指标值的直接比较:

- price_above: `close > indicator`
- price_below: `close < indicator`

条件字典: `{"type": "price_above"}` (无需 threshold)

适用于价格参考线类指标: EMA/SMA/WMA 等均线、BB/Keltner/Donchian 通道轨、PSAR。

---

## 4 Phase 2 - 动态序列/回溯窗口 (4 种)

### 4.1 序列穿越 (cross_above_series / cross_below_series)

检测指标A穿越指标B:

- cross_above_series: `shift(1): A < B` 且 `current: A >= B`
- cross_below_series: `shift(1): A > B` 且 `current: A <= B`

条件字典:
```python
{"type": "cross_above_series", "target_indicator": "EMA", "target_params": {"period": 20}}
```

实现: `_resolve_target_indicator_series()` 从条件字典提取目标指标定义,通过 `_get_indicator_column()` 解析目标列。若目标列不存在,返回全 False。

目标指标选择策略 (`_pick_series_target()`):
1. 优先同类别指标
2. 补充趋势类指标(均线是常见穿越目标)
3. 排除 guard_only 指标
4. 无候选时回退到 "EMA"

### 4.2 回溯窗口 (lookback_any / lookback_all)

在回溯窗口内检查内部条件是否满足:

- lookback_any: `rolling(window).any()` -- 窗口内任一根满足
- lookback_all: `rolling(window).all()` -- 窗口内全部满足

条件字典:
```python
{"type": "lookback_any", "window": 5, "inner": {"type": "price_above"}}
```

`_evaluate_inner_condition()` 支持的内部条件: lt/gt/le/ge, price_above/price_below, 以及通过 `evaluate_condition()` 递归处理的复杂类型。

随机生成: window 从 `[3, 5, 8, 10]` 选择,inner 从 `[price_above, price_below, gt, lt]` 选择。

---

## 5 Phase 3 - 支撑/阻力 (3 种)

### 5.1 触及反弹 (touch_bounce)

价格触及指标线后反弹。三重条件同时满足:

- direction="support": low 触及线 + 收盘在上方 + 下一根收盘更高
- direction="resistance": high 触及线 + 收盘在下方 + 下一根收盘更低

条件字典:
```python
{"type": "touch_bounce", "direction": "support", "proximity_pct": 0.01, "bounce_pct": 0.005}
```

proximity 接近距离 = `abs(low - line) <= abs(line) * proximity_pct`

`core/validation/patterns.py` 中的 `detect_touch_bounce()` 提供更严格的反弹判定,增加了 bounce_pct 最小反弹幅度要求。

### 5.2 角色互换 (role_reversal)

指标线从支撑变阻力或从阻力变支撑:

- role="support": 之前价格在线之下,现在在线之上
- role="resistance": 之前价格在线之上,现在在线之下

条件字典:
```python
{"type": "role_reversal", "role": "resistance", "lookback": 10}
```

实现: 比较 `close.shift(lookback)` 与 `line.shift(lookback)` 的关系 vs 当前的关系。lookback 从 `[5, 10, 15, 20]` 随机选取。

### 5.3 影线触及 (wick_touch)

K线影线触及指标线但收盘在另一侧:

- direction="above": high 触及线 + 收盘在下方 (上影线被拒)
- direction="below": low 触及线 + 收盘在上方 (下影线被拒)

条件字典:
```python
{"type": "wick_touch", "direction": "above", "proximity_pct": 0.01}
```

## 6 Phase 4 - 复杂形态 (2+ 种)

Phase 4 不通过 `executor.py` 的 `evaluate_condition()` 实现,而在 `core/validation/patterns.py` 中独立实现,作为策略验证和形态扫描工具。

### 6.1 顶背离 (divergence_top)

价格创新高但指标未创新高(看跌)。在 lookback 窗口内找价格的局部最高点,取最近两个,判断价格上升但指标下降的模式。

### 6.2 底背离 (divergence_bottom)

价格创新低但指标未创新低(看涨)。与顶背离镜像。

### 6.3 连续方向 (consecutive_up / consecutive_down)

指标连续 N 根严格递增/递减。通过 `_is_strictly_increasing()` / `_is_strictly_decreasing()` 检测,NaN 值不满足。

## 7 指标与条件类型映射

每个指标在 `INDICATOR_REGISTRY` 中定义支持的 `supported_conditions` 列表。

### 7.1 trend 类别

EMA/SMA/WMA/DEMA/TEMA 支持: price_above, price_below, cross_above, cross_below, cross_above_series, cross_below_series。VWAP 仅支持 price_above, price_below。

趋势类指标主要用于价格比较和穿越检测,不使用阈值比较。

### 7.2 momentum 类别

RSI: lt, gt, le, ge。MACD: cross_above, cross_below, gt, lt。Aroon 额外支持 cross_above_series/cross_below_series。其余(Stochastic, CCI, ROC, CMO, TRIX, Williams %R)主要使用阈值和穿越。

### 7.3 volatility 类别

BB: price_above, price_below, lookback_any, lookback_all, touch_bounce, role_reversal, wick_touch。Keltner/Donchian: price_above, price_below, touch_bounce, role_reversal, wick_touch。ATR: gt, lt (guard_only)。

波动率类指标是支撑/阻力条件类型的主要使用者。

### 7.4 volume / trend_strength / structure 类别

- volume: OBV/AD/CVD 使用阈值和穿越; RVOL/VROC 额外支持 lookback; VWMA 同趋势类
- trend_strength: ADX(gt, lt), PSAR(price_above, price_below) -- 均 guard_only
- structure: VolumeProfile(touch_bounce, role_reversal) -- guard_only

## 8 条件字典结构规范

### 8.1 各类型的完整参数

```
Phase 1: {"type": "lt", "threshold": 30}
         {"type": "cross_above", "threshold": 0}
         {"type": "price_above"}

Phase 2: {"type": "cross_above_series", "target_indicator": "EMA",
           "target_params": {"period": 20}}
         {"type": "lookback_any", "window": 5,
           "inner": {"type": "price_above"}}

Phase 3: {"type": "touch_bounce", "direction": "support",
           "proximity_pct": 0.01, "bounce_pct": 0.005}
         {"type": "role_reversal", "role": "resistance", "lookback": 10}
         {"type": "wick_touch", "direction": "above", "proximity_pct": 0.01}
```

### 8.2 必填字段验证

`_validate_condition_structure()` 的验证规则:

| 条件类型 | 必填字段 |
|---------|---------|
| lt/gt/le/ge | threshold |
| cross_above/cross_below | threshold |
| price_above/price_below | 无额外字段 |
| cross_above_series/cross_below_series | target_indicator |
| lookback_any/lookback_all | window, inner |
| touch_bounce | direction |
| role_reversal | role |
| wick_touch | direction |

## 9 条件的随机生成

### 9.1 generate_random_condition()

位于 `core/evolution/operators.py`。从指标的 `supported_conditions` 列表中随机选择类型,然后按类型生成参数:

| 类型 | 参数生成策略 |
|------|------------|
| lt/gt/le/ge | RSI: 预定义列表; 其他: uniform(-1,1) |
| cross_above_series | _pick_series_target() + 默认参数 |
| lookback_any/all | window:[3,5,8,10]; inner:[price_above,price_below,gt,lt] |
| touch_bounce | direction:[support,resistance]; proximity_pct:[0.005,0.01,0.02] |
| role_reversal | role:[support,resistance]; lookback:[5,10,15,20] |
| wick_touch | direction:[above,below]; proximity_pct:[0.005,0.01,0.02] |

回退:注册信息不存在时返回 `{"type": "gt", "threshold": 0.0}`。

## 10 信号角色与 guard_only

### 10.1 四种信号角色

| 角色 | 语义 | 指标限制 |
|------|------|---------|
| entry_trigger | 直接触发买入 | 非 guard_only |
| entry_guard | 过滤入场 | 任何指标 |
| exit_trigger | 直接触发卖出 | 非 guard_only |
| exit_guard | 过滤出场 | 任何指标 |

### 10.2 guard_only 指标

ATR, ADX, PSAR, VolumeProfile 标记为 guard_only,只能用作 guard 角色。指标替换变异中强制执行此约束:

```python
if is_trigger:
    alternatives = [a for a in alternatives
                    if not INDICATOR_REGISTRY[a].guard_only]
```

## 11 指标列解析

### 11.1 列命名规则

`_get_indicator_column()` 按统一规则解析列名:

| 指标 | 列名格式 | 示例 |
|------|---------|------|
| RSI | rsi_{period} | rsi_14 |
| EMA | ema_{period} | ema_20 |
| MACD | macd_{field}_{fast}_{slow}_{signal} | macd_histogram_12_26_9 |
| BB | bb_{field}_{period}_{std} | bb_upper_20_2 |
| Keltner | kc_{ema}_{atr}_{mult}_{field} | kc_20_10_2.0_upper |

多输出指标通过 `field_name` 选择: MACD(macd/signal/histogram), BB(upper/middle/lower), Stochastic(k/d), Aroon(aroon_up/down/osc)。

### 11.2 回退策略

精确列名匹配失败后使用前缀匹配: `[c for c in df.columns if col in c]`。仍失败则抛出 ValueError。

## 12 评估流程总结

### 12.1 单基因评估路径

```
SignalGene -> _get_indicator_column(df, gene) -> indicator_series
           -> evaluate_condition(series, close, condition, df) -> bool Series
           -> fillna(False) -> 按 role 分类收集
```

### 12.2 层内组合

```
all_entry = entry_triggers + entry_guards
entries = combine_signals(all_entry, layer.logic_genes.entry_logic)  # AND/OR
```

### 12.3 MTF 多层组合

```
每层 -> evaluate_layer() -> resample_signals() 到执行时间周期
最终 -> combine_signals(所有层信号, dna.cross_layer_logic)
冲突 -> entry = entry & ~(entry & exit)  # 同时触发时优先 exit
```
